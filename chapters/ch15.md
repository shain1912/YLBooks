## 15.1 마지막 장 — 친구한테 자랑할 수 있는 모양으로

14장까지 오느라 정말 수고 많았습니다. 이 책을 펼쳤던 날을 떠올려 보세요. 키링이 뭔지, `pygame`이 뭔지도 몰랐는데, 지금은 리듬에 맞춰 노트가 떨어지고 판정이 나오는 게임이 손에 쥔 작은 기기에서 돌아가고 있습니다. 이미 그 자체로 대단한 일입니다.

그런데 솔직하게 화면을 다시 한번 봐 주세요. 배경은 새까맣거나 회색이고, 글씨는 기본 폰트라 사각사각한 픽셀 덩어리입니다. "콤보"와 "점수" 자리는 한글이 깨져서 두부처럼 □□로 나올 수도 있습니다. 게임 오버가 되면 그냥 창이 닫힙니다. 결과 화면도 없고, 시작 화면도 없습니다.

동작은 완벽합니다. 딱 5%만 다듬으면 진짜로 자랑할 수 있는 완성품이 됩니다.

이 장에서 할 일은 네 가지입니다.

첫째, 한글 폰트를 제대로 지정해서 글자 깨짐을 없앱니다. 둘째, 다크 배경과 네온 색 팔레트로 화면을 전체적으로 정돈합니다. 셋째, 곡이 끝났을 때 나오는 결과 화면과 S/A/B/C 등급을 추가합니다. 넷째, 시작 화면과 일시정지를 만들어서 친구 앞에서 매끄럽게 시연할 수 있게 합니다.

> **그림 15.1 — "비포·애프터 — 못생긴 화면 vs 다듬은 화면"**
> *스크린샷 2장 좌우 배치. 같은 게임 진행 시점에서 찍은 두 화면. 왼쪽은 기본 폰트·회색 배경·한글 깨짐. 오른쪽은 말굿 폰트·다크 배경·네온 색 UI·한글 정상 출력.*

이 장을 마치면 15.6절에서 친구에게 실제로 도전시켜 볼 것입니다. 그것이 이 책 전체의 최종 보상입니다.

## 15.2 한글 출력 정복하기 — 폰트 명시 지정

pygame을 처음 설치하고 한글을 화면에 출력하면 높은 확률로 글자가 □□□로 뭉개져 나옵니다(그림 15.2). 이것을 개발자 사이에서 흔히 **두부 글자**라고 부릅니다. 한글을 지원하지 않는 폰트가 기본값으로 설정되어 있기 때문입니다.

> **그림 15.2 — "한글이 깨진 화면 — 두부 글자"**
> *스크린샷: pygame 실행 화면. 점수 표시 영역에 "콤보" 자리가 □□로 렌더링된 모습.*

`pygame.font.SysFont("Arial", 24)` 같이 쓰면 pygame이 시스템에서 폰트를 자동으로 찾습니다. 그런데 "Arial"은 영어 전용 폰트라서 한글 문자는 처리할 수 없습니다. 결과로 두부가 나옵니다.

해결책은 `SysFont` 대신 `pygame.font.Font`로 **폰트 파일 경로를 직접 지정**하는 것입니다.

```python
# 이전 — 한글이 깨집니다
font = pygame.font.SysFont("Arial", 24)

# 이후 — 폰트 파일을 직접 지정합니다
font = pygame.font.Font("assets/fonts/Malgun.ttf", 24)
```

> **그림 15.3 — "malgun.ttf 폰트 명시 지정 — 한 줄 수정"**
> *스크린샷: VS Code 편집기. `pygame.font.Font("assets/fonts/Malgun.ttf", 24)` 코드 줄이 노란색으로 강조 표시된 모습.*

이 한 줄로 한글이 정상 출력됩니다(그림 15.4).

> **그림 15.4 — "한글이 정상 출력된 화면"**
> *스크린샷: pygame 화면. "콤보 50 / 점수 24500"이 깔끔하게 표시된 모습.*

**폰트 파일은 어디서 구하나요?**

운영체제마다 경로가 다릅니다(그림 15.5).

> **그림 15.5 — "윈도우 vs 맥 — 폰트 경로 차이"**
> *도식: 왼쪽 Windows 아이콘 + `C:\Windows\Fonts\malgun.ttf` / 오른쪽 macOS 아이콘 + `/System/Library/Fonts/AppleSDGothicNeo.ttc`*

```python
import platform

if platform.system() == "Windows":
    font_path = "C:/Windows/Fonts/malgun.ttf"
elif platform.system() == "Darwin":  # macOS
    font_path = "/System/Library/Fonts/AppleSDGothicNeo.ttc"
else:
    font_path = None  # 리눅스는 따로 설치

if font_path:
    font = pygame.font.Font(font_path, 24)
else:
    font = pygame.font.SysFont("sans-serif", 24)
```

이 방식은 동작은 하지만 **다른 컴퓨터에서 실행하면 경로가 달라져 오류가 납니다.** 더 안전한 방법이 있습니다.

**안전한 방법: 폰트를 프로젝트 폴더에 함께 넣기**

윈도우 `C:\Windows\Fonts\` 폴더에서 `malgun.ttf` 파일을 복사해서 프로젝트 폴더 안 `assets/fonts/` 에 붙여 넣습니다. 이제 어느 컴퓨터에서 실행해도, 심지어 친구 노트북에서도 폰트가 깨지지 않습니다(그림 15.6).

> **그림 15.6 — "안전한 방법 — 프로젝트 폴더에 폰트 같이"**
> *스크린샷: VS Code 사이드바. 프로젝트 폴더 트리에 `assets/fonts/Malgun.ttf` 파일이 추가된 모습.*

```python
# 최종 권장 코드
FONT_PATH = "assets/fonts/Malgun.ttf"
font_large  = pygame.font.Font(FONT_PATH, 48)
font_medium = pygame.font.Font(FONT_PATH, 32)
font_small  = pygame.font.Font(FONT_PATH, 16)
```

폰트 파일 자체는 저작권이 있으므로 재배포 가능한 폰트를 쓰는 것이 좋습니다. 네이버 나눔고딕(`NanumGothic.ttf`)은 무료로 재배포할 수 있어서 실제 배포까지 생각한다면 좋은 선택입니다.

## 15.3 색·UI 다듬기 — 5분 만에 예뻐 보이는 팁

미술을 전공하지 않아도 됩니다. **미리 고른 팔레트를 그대로 쓰면** 게임이 바로 그럴싸해 보입니다.

아래 다섯 가지 색만 씁니다(그림 15.7).

> **그림 15.7 — "네온 색 팔레트 5색"**
> *도식: 색상 견본 5개. #0E0E14(거의 검정 배경) / #FF4D6D(빨간 노트·미스) / #4DFFB4(초록 퍼펙트) / #4DA8FF(파란 굿) / #FFD84D(노란 콤보·점수)*

```python
# 팔레트 상수로 정의해 두기
COLOR_BG      = (14,  14,  20)   # #0E0E14 — 배경
COLOR_NOTE    = (255, 77,  109)  # #FF4D6D — 노트
COLOR_PERFECT = (77,  255, 180)  # #4DFFB4 — 퍼펙트 판정
COLOR_GOOD    = (77,  168, 255)  # #4DA8FF — 굿 판정
COLOR_COMBO   = (255, 216, 77)   # #FFD84D — 콤보·점수
COLOR_WHITE   = (240, 240, 250)  # 거의 흰색 UI 텍스트
```

팔레트를 정했으면 폰트 크기 위계를 설정합니다(그림 15.8).

> **그림 15.8 — "폰트 크기 위계"**
> *도식: 게임 화면 모형 위에 세 영역 표시. 화면 왼쪽 상단 점수 48px / 오른쪽 콤보 32px / 하단 판정 텍스트 16px.*

- **점수**: 48px — 가장 크게. 한눈에 들어와야 합니다.
- **콤보**: 32px — 중간 크기. 중요하지만 점수보다 작아야 집중이 분산되지 않습니다.
- **판정 텍스트 / 키 안내**: 16px — 작게. 보조 정보입니다.

```python
font_score  = pygame.font.Font(FONT_PATH, 48)
font_combo  = pygame.font.Font(FONT_PATH, 32)
font_ui     = pygame.font.Font(FONT_PATH, 16)
```

이제 배경을 채웁니다. 기존 `screen.fill((0, 0, 0))` 을 아래처럼 바꾸면 됩니다.

```python
screen.fill(COLOR_BG)
```

판정 텍스트 색도 분기합니다.

```python
def get_judge_color(judge: str) -> tuple:
    if judge == "PERFECT":
        return COLOR_PERFECT
    elif judge == "GOOD":
        return COLOR_GOOD
    else:  # MISS
        return COLOR_NOTE  # 빨간색

# 판정 표시
color = get_judge_color(last_judge)
```

이 코드를 추가하는 것만으로 각 판정이 다른 색으로 표시됩니다. 퍼펙트를 칠 때마다 초록빛이 번쩍이고, 미스를 내면 빨간 글자가 나타납니다. 시각 피드백이 확실해지면 게임이 훨씬 재미있어집니다.

## 15.4 결과 화면 만들기 — 통계와 등급

지금까지는 곡이 끝나면 그냥 창이 닫히거나 루프가 조용히 종료됩니다. 친구한테 보여줄 때 "여기가 끝이야"라고 말로 설명해야 합니다. 결과 화면이 없으면 게임이 완성된 것 같지 않습니다.

결과 화면에 표시할 내용은 다음 네 가지입니다.

1. **최종 점수**
2. **최대 콤보**
3. **정확도(%)** — PERFECT와 GOOD을 칠 때 마다 누적되고 MISS는 빠집니다
4. **등급** — S / A / B / C

등급 기준은 아래와 같습니다.

| 정확도 | 등급 |
|--------|------|
| 95% 이상 | S |
| 90% 이상 | A |
| 80% 이상 | B |
| 80% 미만 | C |

> **그림 15.9 — "결과 화면 와이어프레임"**
> *도식: 결과 화면 4개 영역. 상단 중앙 큰 등급 문자 / 그 아래 정확도 % / 그 아래 점수·최대 콤보 / 하단 PERFECT·GOOD·MISS 카운트.*

```python
def calculate_grade(accuracy: float) -> str:
    if accuracy >= 95.0:
        return "S"
    elif accuracy >= 90.0:
        return "A"
    elif accuracy >= 80.0:
        return "B"
    else:
        return "C"

def calculate_accuracy(perfect: int, good: int, miss: int) -> float:
    total = perfect + good + miss
    if total == 0:
        return 0.0
    # PERFECT 100%, GOOD 50%, MISS 0%로 가중 계산
    weighted = perfect * 1.0 + good * 0.5 + miss * 0.0
    return (weighted / total) * 100.0

def draw_result_screen(screen, score, max_combo,
                        perfect, good, miss):
    accuracy = calculate_accuracy(perfect, good, miss)
    grade    = calculate_grade(accuracy)

    screen.fill(COLOR_BG)

    # 등급 — 크게
    grade_color = COLOR_PERFECT if grade == "S" else COLOR_COMBO
    surf_grade = font_large.render(grade, True, grade_color)
    screen.blit(surf_grade,
                (SCREEN_W // 2 - surf_grade.get_width() // 2, 80))

    # 정확도
    acc_text = f"정확도  {accuracy:.1f}%"
    surf_acc = font_medium.render(acc_text, True, COLOR_WHITE)
    screen.blit(surf_acc,
                (SCREEN_W // 2 - surf_acc.get_width() // 2, 180))

    # 점수 / 최대 콤보
    surf_score = font_medium.render(f"점수  {score:,}", True, COLOR_COMBO)
    surf_combo = font_medium.render(f"최대 콤보  {max_combo}", True, COLOR_COMBO)
    screen.blit(surf_score,
                (SCREEN_W // 2 - surf_score.get_width() // 2, 240))
    screen.blit(surf_combo,
                (SCREEN_W // 2 - surf_combo.get_width() // 2, 290))

    # 판정 세부 통계
    stats = f"PERFECT {perfect}  /  GOOD {good}  /  MISS {miss}"
    surf_stats = font_ui.render(stats, True, COLOR_WHITE)
    screen.blit(surf_stats,
                (SCREEN_W // 2 - surf_stats.get_width() // 2, 360))

    # 안내
    hint = font_ui.render("R: 재시작   Q: 종료", True, COLOR_WHITE)
    screen.blit(hint,
                (SCREEN_W // 2 - hint.get_width() // 2, 430))

    pygame.display.flip()
```

> **그림 15.10 — "결과 화면 실제 구현 — S 등급"**
> *스크린샷: 결과 화면. 가운데 큰 초록색 "S" 글자 / 정확도 95.2% / PERFECT 152 / GOOD 8 / MISS 0 / 하단 "R: 재시작  Q: 종료".*

게임 루프가 끝나는 시점에 이 함수를 호출하고, R키를 누르면 게임을 다시 시작하도록 연결하면 됩니다 (그림 15.10 참조).

## 15.5 시연 모드 옵션 — 친구에게 보여줄 때 필요한 것들

친구 앞에서 시연할 때 가장 난감한 상황은 두 가지입니다. 하나는 시작하자마자 바로 노트가 쏟아져서 "잠깐, 이게 뭔지 설명해야 하는데"라고 말할 틈이 없는 것입니다. 다른 하나는 도중에 친구가 "잠깐 멈춰봐"라고 하는데 일시정지 버튼이 없는 것입니다.

이 두 가지를 모두 해결하는 화면이 **시작 화면**과 **일시정지 화면**입니다.

**시작 화면**에는 세 가지가 있으면 충분합니다(그림 15.11).

- 곡 선택 목록 (위아래 방향키)
- 난이도 선택 (노트 낙하 속도, 1~5 숫자키)
- 시작 버튼 안내 (스페이스바)

> **그림 15.11 — "시작 화면 — 곡 선택 + 난이도 슬라이더"**
> *스크린샷: 시작 화면. 중앙에 곡 리스트 3개 중 하나가 선택(하이라이트)된 상태. 아래 난이도 1~5 슬라이더 표시. 최하단 "SPACE: 시작" 안내.*

**일시정지 화면**은 P키로 진입하고 뒷배경이 어두워지면서 안내 메시지가 나타납니다(그림 15.12).

> **그림 15.12 — "일시정지(P) 화면"**
> *스크린샷: pygame 화면. 게임 진행 화면 위에 반투명 검정 레이어. 가운데 "PAUSED" + "P: 재개 / R: 재시작 / Q: 종료" 안내문.*

```python
# 키 이벤트 처리 — 게임 루프 안
for event in pygame.event.get():
    if event.type == pygame.KEYDOWN:
        if event.key == pygame.K_p:
            paused = not paused          # P: 토글
        elif event.key == pygame.K_r:
            restart_game()               # R: 재시작
        elif event.key == pygame.K_q:
            pygame.quit()
            sys.exit()

# 일시정지 오버레이
if paused:
    overlay = pygame.Surface((SCREEN_W, SCREEN_H), pygame.SRCALPHA)
    overlay.fill((0, 0, 0, 160))        # 알파 160 — 반투명
    screen.blit(overlay, (0, 0))
    msg = font_medium.render(
        "PAUSED  —  P: 재개 / R: 재시작 / Q: 종료",
        True, COLOR_WHITE)
    screen.blit(msg,
                (SCREEN_W // 2 - msg.get_width() // 2,
                 SCREEN_H // 2 - msg.get_height() // 2))
    pygame.display.flip()
    continue                             # 게임 로직 건너뜀
```

이 코드를 시작 화면·결과 화면과 함께 한 번에 추가하려면 Antigravity에 한꺼번에 요청하면 됩니다(그림 15.13).

> **그림 15.13 — "Antigravity로 시작·결과·일시정지 화면 한 번에 추가"**
> *스크린샷: Antigravity 채팅창. "시작 화면(곡 선택·난이도), 결과 화면(S/A/B/C 등급·정확도), 일시정지(P키 반투명 오버레이)를 한꺼번에 추가해 줘. 기존 game.py의 게임 루프와 연결해서 …" 로 시작하는 긴 프롬프트가 입력된 모습.*

Antigravity에 맥락을 충분히 줄수록 한 번에 잘 나옵니다. "기존 파일에서 `game_loop()` 함수가 어디서 끝나는지"와 "점수·콤보·PERFECT·GOOD·MISS 변수 이름"을 프롬프트에 포함하세요.

## 15.6 발표 시연 — 키링 + 노트북 + 친구

이제 실제로 발표 시연을 해 봅시다.

준비물은 단 세 가지입니다.

1. 키링 (게임기)
2. 노트북 (게임 화면 출력)
3. 친구 한 명 이상

책상 위에 키링을 놓고 노트북을 옆에 세웁니다. 게임 화면이 친구를 향하도록 각도를 맞춥니다. 키링의 USB 케이블은 노트북에 꽂혀 있습니다.

> **그림 15.14 — "친구가 키링을 잡고 도전하는 모습"**
> *촬영: 흰 책상 위. 노트북 화면에 게임 진행 중. 학생 두 명 — 한 명이 키링을 양손으로 쥐고 있고, 다른 한 명은 옆에서 화면을 바라보는 자세. 자연광 + 데스크 램프. 얼굴 작게 또는 옆모습.*

**발표 시연 흐름**

발표 시연의 한 사이클은 다음 네 단계입니다(그림 15.15).

1. **시작 화면** — 곡을 고르고 난이도를 설정합니다. 스페이스바를 눌러 시작합니다.
2. **게임 진행** — 친구가 키링 버튼으로 노트를 판정합니다.
3. **곡 끝** — 마지막 노트가 지나가면 자동으로 결과 화면으로 넘어갑니다.
4. **결과 화면** — 점수·정확도·등급이 표시됩니다.

> **그림 15.15 — "발표 시연 한 사이클 (4컷 시퀀스)"**
> *스크린샷 4장 가로 나열: ① 시작 화면(곡 선택) → ② 게임 진행(노트 낙하) → ③ 곡 끝 순간(마지막 노트) → ④ 결과 화면 S 등급.*

현장에서 자주 쓰이는 멘트입니다.

- "이 버튼 네 개가 레인이야. 노트가 떨어지면 맞춰서 눌러봐."
- "P 누르면 잠깐 멈춰. 먼저 화면 좀 보고 해도 돼."
- "결과 화면에서 S가 나오면 잘 한 거야. 한번 해봐."

발표 자리라면 노트북 화면을 강의실 빔 프로젝터에 연결하거나, 미러링으로 큰 화면에 띄우면 청중 전체가 같이 볼 수 있습니다.
