# 5장 Arduino IDE 설정과 첫 업로드

> **이 장에서 배울 것**
>
> 4장에서 키링 보드를 컴퓨터에 꽂아 장치 인식까지 확인했습니다. 이제 진짜 코드를 보드 안에 집어넣을 차례입니다. 5장에서는 Arduino IDE를 설치하고, 보드와 포트를 올바르게 설정한 뒤, 단 한 줄의 코드 변경만으로 LED 깜빡임 속도가 바뀌는 경험을 합니다. "내 코드가 진짜 칩 위에서 돌고 있다"는 감각을 처음으로 느끼는 순간입니다.

## 5.1 Arduino IDE — 펌웨어를 굽는 도구

여러분이 작성한 C/C++ 코드는 사람이 읽기 좋은 텍스트입니다. ATmega32U4는 이 텍스트를 그대로 이해하지 못합니다. 칩이 알아듣는 것은 오직 0과 1로 구성된 **기계어(machine code)** 뿐입니다. Arduino IDE는 이 변환을 자동으로 처리해 주는 도구입니다.

작업 흐름을 한 문장으로 정리하면 이렇습니다. 여러분이 `.ino` 파일에 코드를 작성하면, IDE가 이를 컴파일해 `.hex` 파일(기계어 묶음)로 만들고, 그것을 USB 케이블을 통해 보드로 전송합니다. 보드는 받은 내용을 플래시 메모리에 씁니다. USB를 뽑은 뒤에도 코드가 사라지지 않는 이유가 바로 여기에 있습니다 (그림 5.1 참조).

> **그림 5.1 — "코드가 펌웨어가 되어 보드로 가는 과정"**
>
> *[일러스트: .ino → 컴파일 → .hex → USB → ATmega32U4 플래시. 4단계 화살표. 각 단계 아래 짧은 설명 텍스트 포함.]*

이처럼 보드 안에 저장되어 하드웨어를 직접 제어하는 소프트웨어를 **펌웨어(firmware)** 라고 부릅니다. 스마트폰 앱처럼 운영체제 위에서 돌아가는 것이 아니라, 칩이 켜지는 순간부터 직접 실행됩니다. 소프트웨어보다 하드웨어에 가까이 붙어 있다는 뜻에서 "펌(firm, 단단한)" 웨어라는 이름이 붙었습니다. 이번 교재에서 만들 키링 게임 프로그램도 모두 펌웨어 형태로 보드에 올라갑니다.

## 5.2 Arduino IDE 2.x 설치

Arduino IDE 2.x는 공식 사이트에서 무료로 내려받을 수 있습니다. 브라우저를 열고 `arduino.cc/en/software` 로 이동합니다 (그림 5.2 참조).

> **그림 5.2 — "Arduino 공식 페이지 다운로드"**
>
> *[스크린샷: arduino.cc/en/software 페이지. "Arduino IDE 2" 섹션의 Windows Installer 버튼과 macOS 버튼에 각각 빨간 화살표 표시.]*

페이지 중앙에 **Arduino IDE 2** 섹션이 보입니다. 자신의 운영체제에 맞는 버튼을 클릭합니다.

- **Windows**: `Windows Win 10 and newer, 64 bits` 버튼 → `.exe` 인스톨러 다운로드
- **macOS**: `macOS 10.15 "Catalina" or newer, 64 bits` 버튼 → `.dmg` 파일 다운로드

다운로드가 완료되면 인스톨러를 실행합니다. macOS는 `.dmg`를 열어 아이콘을 Applications 폴더로 드래그하면 설치가 끝납니다.

Windows 사용자는 한 가지 중요한 단계가 있습니다. 설치 중간에 **"Arduino USB Driver를 설치하겠습니까?"** 라는 보안 경고 창이 등장합니다 (그림 5.3 참조). 반드시 **설치(Install)** 를 클릭해야 합니다. 이 드라이버가 없으면 Windows가 보드를 인식하지 못해 업로드가 불가능합니다.

> **그림 5.3 — "윈도우 설치 중 드라이버 설치 동의 창"**
>
> *[스크린샷: "Arduino USB Driver" Windows 보안 경고 대화상자. "이 장치 소프트웨어를 설치하시겠습니까?" 메시지. 설치 버튼에 빨간 박스.]*

설치가 끝나면 바탕화면의 Arduino 아이콘을 더블클릭해 IDE를 엽니다 (그림 5.4 참조). 왼쪽에 아이콘 사이드바가 있고, 중앙에 빈 코드 편집창이 열립니다. 이 화면이 보이면 설치는 완료된 것입니다.

> **그림 5.4 — "Arduino IDE 2.x 첫 실행 화면"**
>
> *[스크린샷: Arduino IDE 2 초기 화면. 좌측 사이드바 아이콘들(폴더, 보드 매니저, 라이브러리 매니저, 시리얼 모니터)과 중앙 빈 스케치 편집 영역 전체.]*

## 5.3 보드 매니저로 ATmega32U4 추가

Arduino IDE는 수십 종의 보드를 지원합니다. 보드마다 칩 사양과 컴파일 방식이 다르기 때문에, 사용할 보드에 맞는 **보드 패키지**를 먼저 설치해야 합니다.

왼쪽 사이드바에서 회로 기판 모양 아이콘을 클릭하거나, 메뉴 **도구(Tools) → 보드(Board) → 보드 매니저(Boards Manager)** 를 선택합니다 (그림 5.5 참조).

> **그림 5.5 — "보드 매니저 열기"**
>
> *[스크린샷: Arduino IDE 2 좌측 사이드바. 보드 매니저 아이콘에 빨간 원 표시. 또는 메뉴 도구 → 보드 → 보드 매니저 경로 하이라이트.]*

검색창에 `AVR` 을 입력합니다. **Arduino AVR Boards** 항목이 나타납니다. Arduino IDE를 처음 설치하면 이 패키지가 기본으로 포함되어 있어서 이미 `INSTALLED` 상태일 수 있습니다 (그림 5.6 참조). `INSTALLED` 표시가 없다면 **설치(Install)** 버튼을 클릭합니다. Arduino Leonardo와 ATmega32U4 지원이 여기에 들어 있습니다.

> **그림 5.6 — "Arduino AVR Boards 설치 (기본 포함)"**
>
> *[스크린샷: 보드 매니저 패널. 검색창에 "AVR" 입력. "Arduino AVR Boards" 항목 옆에 "INSTALLED" 초록 라벨 또는 설치 버튼.]*

**SparkFun Pro Micro를 사용하는 경우**: Leonardo 대신 Pro Micro 보드를 사용한다면 SparkFun 전용 패키지가 필요합니다. 메뉴 **파일(File) → 환경설정(Preferences)** 을 열고, **추가 보드 매니저 URL** 입력란에 아래 주소를 붙여 넣습니다 (그림 5.7 참조).

```
https://raw.githubusercontent.com/sparkfun/Arduino_Boards/main/IDE_Board_Manager/package_sparkfun_index.json
```

확인을 누른 뒤 보드 매니저에서 `SparkFun` 을 검색하면 **SparkFun AVR Boards** 패키지가 나타납니다. 설치 후 5.4절로 이동합니다.

> **그림 5.7 — "환경설정에 SparkFun 보드 URL 추가"**
>
> *[스크린샷: 파일 → 환경설정 대화상자. "추가 보드 매니저 URLs" 입력 필드에 SparkFun URL이 입력된 상태. 확인 버튼에 빨간 박스.]*

## 5.4 보드와 포트 선택

보드 패키지 설치가 끝났으면, IDE에게 "지금 어떤 보드를 쓰는지"와 "어느 USB 포트에 꽂혀 있는지"를 알려줘야 합니다.

### 보드 선택

메뉴 **도구(Tools) → 보드(Board) → Arduino AVR Boards** 를 펼칩니다. 목록에서 보드 종류를 선택합니다 (그림 5.8 참조).

- 키링이 **Arduino Leonardo** 기반이라면 → **Arduino Leonardo** 선택
- 키링이 **SparkFun Pro Micro** 기반이라면 → **SparkFun Pro Micro** 선택

어떤 보드인지 모를 경우 제품 패키지나 키링 PCB 뒷면 실크스크린(흰 글씨) 확인합니다.

> **그림 5.8 — "보드 선택 — Arduino Leonardo"**
>
> *[스크린샷: 도구 → 보드 → Arduino AVR Boards 서브메뉴. "Arduino Leonardo" 항목에 체크 표시 또는 빨간 박스.]*

### 포트 선택

메뉴 **도구(Tools) → 포트(Port)** 를 클릭합니다. 4장에서 장치 관리자(또는 터미널)로 확인해 둔 COM 번호(예: `COM5`)를 선택합니다 (그림 5.9 참조). Mac에서는 `/dev/cu.usbmodem...` 형태의 항목을 선택합니다.

> **그림 5.9 — "포트 선택 — COM5"**
>
> *[스크린샷: 도구 → 포트 서브메뉴. "COM5 (Arduino Leonardo)" 항목에 체크 표시.]*

포트 목록이 비어 있다면 USB 케이블을 다시 꽂거나, 4장으로 돌아가 장치 인식 여부를 재확인합니다.

## 5.5 첫 코드 — Blink 예제 열기

설정이 끝났으니 코드를 올릴 차례입니다. 처음이라 직접 작성하기보다는, IDE에 내장된 **Blink 예제**를 사용합니다. Blink는 보드의 내장 LED를 1초 간격으로 껐다 켰다 반복하는 가장 간단한 프로그램으로, 아두이노 세계의 "Hello, World"입니다.

메뉴 **파일(File) → 예제(Examples) → 01.Basics → Blink** 를 선택합니다 (그림 5.10 참조). 새 창에 코드가 열립니다.

> **그림 5.10 — "Blink 예제 열기 경로"**
>
> *[스크린샷: 파일 → 예제 → 01.Basics → Blink 메뉴 경로 전체. Blink 항목에 빨간 박스.]*

열린 코드는 다음과 같습니다 (그림 5.11 참조).

```cpp
// 내장 LED가 연결된 핀 번호 (보드마다 다를 수 있음)
// Arduino Leonardo / Pro Micro는 보통 핀 13 또는 LED_BUILTIN 사용

void setup() {
  // 딱 한 번 실행되는 초기화 구역
  pinMode(LED_BUILTIN, OUTPUT); // 내장 LED 핀을 "출력" 모드로 설정
}

void loop() {
  // 전원이 켜져 있는 동안 무한 반복
  digitalWrite(LED_BUILTIN, HIGH); // LED 켜기 (핀에 5V 출력)
  delay(1000);                     // 1000밀리초(= 1초) 대기
  digitalWrite(LED_BUILTIN, LOW);  // LED 끄기 (핀에 0V 출력)
  delay(1000);                     // 1초 대기
}
```

> **그림 5.11 — "Blink 코드 전체 — 한 줄씩 주석"**
>
> *[스크린샷: Blink 코드가 열린 Arduino IDE 편집창. setup/loop/pinMode/digitalWrite/delay 각 부분에 한국어 주석 라벨 오버레이 화살표 표시.]*

코드 구조를 한 줄씩 살펴보겠습니다.

- **`void setup()`**: 보드 전원이 켜지거나 리셋될 때 **딱 한 번** 실행됩니다. 초기 설정 코드(핀 모드 설정, 시리얼 통신 시작 등)를 여기에 씁니다.
- **`void loop()`**: `setup()`이 끝나면 이 함수가 **무한 반복** 됩니다. 전원이 끊기지 않는 한 계속 돌아갑니다.
- **`pinMode(핀, 모드)`**: 특정 핀이 입력(INPUT)인지 출력(OUTPUT)인지 선언합니다. `LED_BUILTIN`은 해당 보드의 내장 LED 핀 번호를 자동으로 가리키는 상수입니다.
- **`digitalWrite(핀, 값)`**: 핀에 `HIGH`(5V) 또는 `LOW`(0V)를 출력합니다. HIGH면 LED에 전류가 흘러 켜지고, LOW면 전류가 끊겨 꺼집니다.
- **`delay(밀리초)`**: 지정한 시간(1ms = 0.001초) 동안 아무것도 하지 않고 대기합니다. `delay(1000)`은 1초 대기입니다.

이 다섯 가지 개념만 알아도 앞으로 나올 예제 대부분의 뼈대를 읽을 수 있습니다.

## 5.6 컴파일과 업로드 두 단계

IDE 상단 툴바 왼쪽에 버튼 두 개가 나란히 있습니다 (그림 5.12 참조). 이 둘은 완전히 다른 작업을 수행하며, 역할을 혼동하면 에러가 생겼을 때 원인을 찾기 어렵습니다.

> **그림 5.12 — "컴파일 버튼(체크)과 업로드 버튼(화살표) 위치"**
>
> *[스크린샷: Arduino IDE 2 툴바. 체크(✓) 아이콘과 오른쪽 화살표(→) 아이콘 각각에 빨간 박스. 각 버튼 아래 "컴파일", "업로드" 라벨 오버레이.]*

### 체크 버튼 — 컴파일

**체크(✓) 버튼**을 클릭하면 코드를 `.hex` 기계어로 변환하는 **컴파일**만 수행합니다. 보드로 전송하지 않습니다. 코드에 문법 오류가 있는지 빠르게 확인할 때 씁니다 (그림 5.13 참조). 콘솔 창에 `Compiling sketch...` 메시지가 흐르다가 성공하면 `Compilation successful.` 이 출력됩니다.

> **그림 5.13 — "컴파일 진행 중 콘솔 출력"**
>
> *[스크린샷: Arduino IDE 2 하단 콘솔 영역. "Compiling sketch..." 텍스트가 표시된 상태.]*

### 화살표 버튼 — 업로드

**화살표(→) 버튼**을 클릭하면 컴파일 후 즉시 **보드에 전송(업로드)** 까지 수행합니다. 업로드 중 IDE는 보드에 리셋 신호를 자동으로 보내 부트로더가 실행되도록 합니다. 이 덕분에 별도의 조작 없이 업로드가 가능합니다.

업로드가 성공하면 콘솔에 초록색으로 **`Done uploading.`** 메시지가 표시됩니다 (그림 5.14 참조). 이 메시지가 보이는 순간, 여러분의 코드는 ATmega32U4 플래시 메모리 안에 영구 기록된 것입니다.

> **그림 5.14 — "업로드 성공 메시지"**
>
> *[스크린샷: Arduino IDE 2 하단 콘솔. "Done uploading." 초록색 텍스트. 그 위로 용량 정보(스케치 크기, 메모리 사용량) 텍스트 함께 보임.]*

> **핵심 구분 요약**: 체크 버튼은 "오류 검사"이고 화살표 버튼은 "오류 검사 + 보드에 굽기"입니다. 최종 결과물을 올릴 때는 반드시 화살표 버튼을 눌러야 합니다.

## 5.7 LED 깜빡임 확인

업로드가 끝나면 보드를 바라봅니다. 키링 PCB 위 작은 LED 하나가 **1초 켜지고 1초 꺼지기를 반복**하고 있을 것입니다 (그림 5.15 참조).

> **그림 5.15 — "보드의 내장 LED가 깜빡이는 모습"**
>
> *촬영 지시: 약간 어두운 책상 위. PCB 키링 전체가 화면 안에 들어오도록 top-down 또는 30도 앵글. LED가 켜진 순간을 캡처. 노출 약 0.5초, LED 빛 번짐이 자연스럽게 표현되도록.*

이 깜빡임이 사소해 보일 수 있습니다. 하지만 이 순간 일어난 일을 생각해 보면 결코 작은 일이 아닙니다. 여러분이 텍스트 편집기에서 입력한 코드가, 기계어로 번역되고, USB를 통해 전기 신호로 흘러가서, 손톱만 한 칩 안에 새겨졌습니다. 그리고 지금 그 코드가 1초마다 `HIGH`와 `LOW`를 번갈아 내보내면서 LED를 직접 켰다 끄고 있습니다. 칩은 여러분의 코드대로 정확히 움직이고 있습니다.

이제 코드를 살짝 수정해 보겠습니다. `loop()` 함수 안의 `delay(1000)` 을 **`delay(100)`** 으로 두 곳 모두 바꿉니다. 그리고 화살표 버튼으로 다시 업로드합니다.

```cpp
void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  delay(100);   // 1000에서 100으로 변경
  digitalWrite(LED_BUILTIN, LOW);
  delay(100);   // 1000에서 100으로 변경
}
```

업로드가 끝나면 LED가 이전보다 **10배 빠르게** 깜빡입니다 (그림 5.16 참조). 숫자 하나만 바꿨는데 눈에 보이는 물리적 결과가 달라졌습니다. 이것이 펌웨어 프로그래밍의 직접성입니다. 앱 개발과 달리 운영체제나 프레임워크 없이, 코드가 하드웨어에 곧장 닿습니다.

> **그림 5.16 — "delay(100)으로 바꾼 후 빠른 점멸"**
>
> *촬영 지시: 그림 5.15와 같은 구도. 노출 시간을 1초로 길게 설정. 빠른 LED 점멸이 가로 줄무늬 또는 여러 번의 빛 흔적으로 시각화되도록. 빠른 점멸과 느린 점멸의 차이를 한눈에 비교 가능하게.*

## 5.8 자주 만나는 에러 3가지

처음 업로드할 때 에러가 뜨더라도 당황하지 않아도 됩니다. 대부분의 문제는 포트 설정이나 보드 점유 같은 단순한 원인에서 비롯됩니다. 가장 자주 만나는 에러 세 가지와 해결법을 정리합니다 (그림 5.17 참조).

> **그림 5.17 — "에러 메시지 3종 비교"**
>
> *[스크린샷: 세 가지 에러 메시지를 세로로 합성한 이미지. 각 에러 텍스트 옆에 빨간 박스와 한 줄 원인 라벨 오버레이.]*

| 에러 메시지 | 원인 | 해결 방법 |
|---|---|---|
| `avrdude: ser_open(): can't open device` | 포트 번호가 잘못 선택됨 | 도구 → 포트에서 올바른 COM 번호 선택 |
| `Error opening serial port 'COMX'` | 다른 프로그램(시리얼 모니터, 게임 소프트웨어 등)이 해당 포트를 점유 중 | 점유 중인 프로그램 종료 후 재시도 |
| 업로드 진행 바가 멈추고 무한 대기 | 보드가 부트로더 진입에 실패 | 업로드 시작 직후 리셋 버튼을 빠르게 두 번 탭(더블탭 리셋) |

세 번째 에러인 업로드 무한 대기는 조금 더 설명이 필요합니다. 화살표 버튼을 눌렀을 때 콘솔에 `Uploading...` 이 뜬 채로 멈추면, 보드를 수동으로 부트로더 모드에 진입시켜야 합니다. 리셋 버튼을 약 0.5초 간격으로 **두 번 빠르게** 누릅니다(더블탭). 보드의 상태 LED가 천천히 맥박치듯 켜졌다 꺼지면 부트로더 모드 진입 성공입니다. 이 상태에서 곧바로 업로드 버튼을 다시 누릅니다 (그림 5.18 참조).

> **그림 5.18 — "포트 선택 잘못했을 때 해결 흐름"**
>
> *[일러스트: 에러 발생 → 도구 → 포트 재확인 → 올바른 COM 번호 선택 → 업로드 재시도. 4단계 오른쪽 방향 화살표 흐름도. 각 단계 아이콘과 짧은 설명 텍스트.]*

세 가지 에러 중 어느 것도 코드 자체의 문제가 아닙니다. 환경 설정(포트, 보드 선택, 프로그램 점유)을 하나씩 점검하면 대부분 해결됩니다. 해결이 안 된다면 Claude Code에 에러 메시지 전체를 붙여 넣어 물어보세요. 정확한 메시지를 그대로 전달하는 것이 빠른 해결의 핵심입니다.

## 5.9 Arduino CLI — 터미널로 빠르게 반복하기

지금까지는 Arduino IDE의 GUI(그래픽 화면)에서 업로드 버튼을 눌렀습니다. IDE는 친절하지만, 코드 한 줄 고치고 화살표 버튼을 누르고 콘솔이 끝나기를 기다리는 흐름이 6장 이후 반복되면 의외로 답답합니다. **Arduino CLI**는 같은 일을 터미널 명령 한 줄로 끝낼 수 있게 해 주는 공식 도구입니다. Claude Code 터미널 안에서 그대로 실행할 수 있으니, AI 페어 코딩 흐름과도 잘 맞습니다.

> **그림 5.19 — "GUI(IDE)와 CLI 같은 일 두 방식"**
>
> *[일러스트: 좌측 IDE 창의 컴파일·업로드 화살표 버튼 / 우측 터미널의 `arduino-cli compile` `arduino-cli upload` 두 줄. 가운데 양방향 화살표 + "같은 결과" 라벨.]*

### 설치

윈도우에서는 `winget`이 가장 간단합니다.

```bash
winget install ArduinoSA.CLI
```

`winget`이 없거나 Mac/Linux라면 공식 설치 스크립트를 씁니다.

```bash
# macOS / Linux / WSL
curl -fsSL https://raw.githubusercontent.com/arduino/arduino-cli/master/install.sh | sh
```

설치 후 새 터미널에서 `arduino-cli version` 으로 확인합니다 (그림 5.20 참조).

> **그림 5.20 — "arduino-cli 설치 확인"**
>
> *[스크린샷: 터미널에 `arduino-cli version` 입력 후 `arduino-cli Version: 1.x.x` 형태의 출력. 버전 번호 빨간 박스.]*

### 첫 설정

처음 한 번만 실행하면 되는 초기화 명령 두 줄이 있습니다.

```bash
arduino-cli config init                          # 설정 파일 생성
arduino-cli core install arduino:avr             # ATmega32U4용 보드 패키지 설치
```

`arduino-cli board list` 로 연결된 보드를 확인할 수 있습니다. 키링이 꽂혀 있다면 `COM5 Arduino Leonardo` 같은 줄이 보입니다 (그림 5.21 참조).

> **그림 5.21 — "arduino-cli board list — 키링 인식 결과"**
>
> *[스크린샷: 터미널 `arduino-cli board list` 실행 결과. Port·Type·Board Name·FQBN 컬럼이 표 형식으로 출력. Arduino Leonardo 줄에 노란 강조.]*

### 컴파일과 업로드

5.5절의 Blink 스케치 폴더(`Blink/`)에 들어가서 두 명령을 차례로 실행합니다.

```bash
# 컴파일
arduino-cli compile --fqbn arduino:avr:leonardo Blink

# 업로드
arduino-cli upload -p COM5 --fqbn arduino:avr:leonardo Blink
```

`--fqbn` 은 "Fully Qualified Board Name"의 줄임으로, 어떤 보드용으로 컴파일할지 알려 주는 식별자입니다. Pro Micro 호환이라면 `arduino:avr:leonardo` 또는 `SparkFun:avr:promicro` 를 씁니다. 포트 번호 `COM5` 는 자기 환경의 번호로 바꿉니다.

이 두 명령을 한 줄로 묶어 두면 코드 수정 후 재업로드가 즉시 됩니다.

```bash
arduino-cli compile --fqbn arduino:avr:leonardo Blink && \
  arduino-cli upload -p COM5 --fqbn arduino:avr:leonardo Blink
```

### 시리얼 모니터도 터미널에서

7장에서 디바운싱을 디버깅할 때 IDE의 시리얼 모니터를 열게 됩니다. CLI에도 같은 기능이 있습니다.

```bash
arduino-cli monitor -p COM5 -c baudrate=9600
```

화면 가득 출력이 흐르면 `Ctrl + C`로 종료합니다.

### 언제 IDE를 쓰고 언제 CLI를 쓰나

| 상황 | 도구 |
|---|---|
| 처음 보드 인식·드라이버 점검·예제 둘러보기 | Arduino IDE |
| 코드 1~2줄 고치고 빠르게 재업로드 (6~8장 반복) | Arduino CLI |
| Claude Code 안에서 AI에게 빌드까지 시킬 때 | Arduino CLI |
| 시리얼 플로터로 그래프 보고 싶을 때 | Arduino IDE |

이 책의 6장부터는 코드 수정 → 재업로드 사이클이 잦아집니다. CLI를 한 번 익혀 두면 그 사이클이 1/3 시간으로 줄어듭니다. 처음부터 부담 갖지 마시고, IDE로 시작했다가 손에 익으면 CLI로 갈아타도 충분합니다.

---

> **5장 정리**
>
> - Arduino IDE는 C/C++ 코드를 기계어(.hex)로 컴파일하고 USB로 보드 플래시 메모리에 기록하는 도구입니다.
> - 설치 후 보드 패키지를 추가하고, 보드 종류와 포트 번호를 정확히 선택해야 업로드가 됩니다.
> - 체크 버튼(컴파일)과 화살표 버튼(컴파일 + 업로드)은 다른 동작입니다.
> - Blink 예제로 첫 업로드를 성공시키고, delay 값을 바꿔 코드 변경이 하드웨어에 즉각 반영됨을 확인했습니다.
> - 업로드 에러 3종(포트 오류, 포트 점유, 부트로더 진입 실패)은 환경 설정 점검으로 해결할 수 있습니다.
>
> 6장에서는 4개의 버튼 입력을 읽어서 화면에 반응시키는 코드를 작성합니다.
