# 4장 키링 보드 처음 만나기

> **이 장에서 배울 것**
>
> 지금까지 도구를 설치하고 AI와 대화하는 법을 익혔습니다. 이제 드디어 실물 하드웨어를 꺼낼 차례입니다. 4장은 키링 PCB 보드를 손에 들고 각 부위가 무엇인지 파악한 뒤, USB 케이블 하나로 컴퓨터에 연결해 "장치 인식"이 뜨는 순간까지 이끌어 드립니다. 코드는 한 줄도 없습니다. 그냥 꽂아 보는 것만으로도 오늘의 목표는 완성입니다.

---

## 4.1 손바닥만 한 컴퓨터 — ATmega32U4

택배 상자를 열면 제일 먼저 눈에 들어오는 건 엄지손톱보다 조금 큰 초록색(또는 검정색) PCB 한 장입니다. 무게는 5g도 안 됩니다. 그런데 이 얇은 판 위에는 완전한 컴퓨터가 들어 있습니다. 바로 **ATmega32U4** 칩 덕분입니다 (그림 4.2 참조).

> **그림 4.1 — "PCB 키링 보드 — 동전과 크기 비교"**
>
> *촬영 지시: 흰 A4 용지 위 top-down 앵글. 키링 PCB 보드와 500원 동전을 나란히 배치. 자연광 또는 데스크 램프. 그림자 최소화.*

ATmega32U4는 마이크로컨트롤러라는 종류의 칩입니다. 쉽게 말하면, CPU·메모리·I/O가 손톱만 한 패키지 하나에 전부 들어간 "올인원 소형 컴퓨터"입니다. 아두이노 레오나르도(Arduino Leonardo)와 프로 마이크로(Pro Micro)에도 똑같이 쓰이는 칩이라, 인터넷에 예제가 매우 많습니다.

> **그림 4.2 — "ATmega32U4 칩 클로즈업"**
>
> *촬영 지시: PCB 메인 칩을 매크로 렌즈로 클로즈업. "ATMEGA32U4" 각인이 뚜렷이 보이도록 핀트 맞춤. 흰 배경.*

이 칩의 핵심 사양을 간단히 보면 (그림 4.4 참조):

- **플래시 메모리 32KB** — 우리가 나중에 올릴 펌웨어 코드가 여기 저장됩니다. USB를 뽑아도 지워지지 않습니다.
- **SRAM 2.5KB** — 실행 중에 변수가 저장되는 공간입니다. 이 값이 작아서 메모리 절약에 주의해야 할 때가 간혹 있습니다.
- **클럭 16MHz** — 초당 1600만 번 연산합니다. 숫자는 작아 보여도, 버튼 입력 처리나 HID 통신에는 충분합니다.
- **USB Full Speed 내장** — 별도 USB 변환 칩 없이 USB를 직접 처리합니다. 이 덕분에 키보드/게임패드 흉내가 가능합니다.

> **그림 4.4 — "주요 사양 표"**
>
> *[표: 칩 ATmega32U4 / 플래시 32KB / SRAM 2.5KB / 클럭 16MHz / USB Full Speed / 가격 1.5~2만원]*

"플래시가 32KB밖에 안 되면 너무 작은 거 아닌가?"라고 걱정할 수 있습니다. 걱정 마세요. 우리가 만들 4버튼 HID 펌웨어는 컴파일하면 5~8KB 안팎입니다. 공간은 넉넉합니다. 이 작은 칩 하나가 오늘 우리 키링의 두뇌가 됩니다.

## 4.2 USB HID — 컴퓨터가 키링을 키보드로 받아들이는 마법

"HID"라는 단어가 낯설 수 있습니다. **Human Interface Device**, 즉 사람이 컴퓨터에 명령을 전달하는 장치를 가리키는 표준 규격입니다. 마우스·키보드·게임패드가 모두 HID입니다. 이 규격의 가장 좋은 점은 **드라이버를 따로 깔 필요가 없다**는 것입니다. 윈도우·macOS·Linux 모두 USB에서 HID 신호를 감지하면 "아, 이거 입력 장치네?" 하고 자동으로 알아서 받아들입니다. USB로 꽂는 순간 운영체제가 먼저 악수를 청하고, 우리 키링이 "저는 키보드입니다"라고 답하면 끝입니다. 별도 드라이버, 별도 설정이 없습니다. 우리가 5장부터 올릴 펌웨어가 바로 이 HID 표준을 적극 활용합니다.

> **그림 4.3 — "USB HID 개념도"**
>
> *[일러스트: 키링 → USB 케이블 → 컴퓨터 순서로 화살표 연결. 컴퓨터 안에 "이건 키보드구나!" 말풍선. 중간에 "HID 표준" 박스 표시]*

## 4.3 키링 한 바퀴 둘러보기 — PCB 부위별 명칭

PCB를 손에 들고 앞면을 바라봅니다. 눈에 띄는 부위를 하나씩 짚어 보겠습니다 (그림 4.5 참조). 참고로, 이 부품들을 직접 납땜할 일은 없습니다. 다 박혀서 나옵니다.

> **그림 4.5 — "PCB 앞면 부위별 명칭"**
>
> *촬영 지시: 보드 앞면을 정수직 앵글로 촬영. 후작업으로 USB-C 포트, 리셋 버튼, 버튼 4개, LED, 칩 위치에 화살표와 라벨 오버레이. 흰 배경.*

**① USB-C 포트** — 보드 한쪽 끝(보통 하단)에 달린 납작한 포트입니다. 전원 공급과 데이터 통신을 동시에 담당합니다. 케이블을 꽂는 방향이 없어서(앞뒤 구분 없음) 편리합니다.

**② 리셋 버튼** — 작은 택타일 스위치입니다. 누르면 칩이 재시작됩니다. 4.6절에서 설명할 "더블탭 리셋 트릭"의 주인공입니다.

**③ 입력 버튼 4개** — 게임에서 A / B / C / D 역할을 할 버튼들입니다. PCB에 이미 SMD 방식으로 실장(또는 끼워진 상태)되어 있습니다. 펌웨어에서는 D2~D5 핀으로 읽을 예정입니다 (그림 4.7 참조).

**④ 상태 LED** — 전원이 들어오면 켜지고, 부트로더 모드에서는 빠르게 깜빡입니다. 보드가 살아있는지 한눈에 확인하는 신호등 역할입니다.

**⑤ ATmega32U4 칩** — 4.1절에서 설명한 두뇌입니다. 보드 중앙에 검은 사각 패키지로 자리 잡고 있습니다.

**⑥ 조이스틱 모듈** — 보드 종류에 따라 있을 수 있습니다. 있으면 아날로그 스틱이 PCB에 함께 달려 있습니다. 16장에서 본격적으로 다룹니다. 지금은 버튼 4개에만 집중합니다.

> **그림 4.6 — "PCB 뒷면"**
>
> *촬영 지시: 보드 뒷면을 정수직 매크로 앵글로 촬영. 표면실장 부품이 뚜렷이 보이도록. 흰 배경.*

> **그림 4.7 — "Pro Micro 호환 핀맵 다이어그램"**
>
> *[일러스트: 표준 Pro Micro 핀맵. GPIO 번호와 특수기능(SDA, SCL, RX, TX) 표시. 우리 키링의 버튼 4개가 D2~D5에 매핑된다고 가정하고, 그 4핀을 색상 강조]*

## 4.4 처음 USB로 꽂아보기 — 인식되나?

이제 실제로 연결해 봅니다. USB-C 케이블을 준비하는데, 한 가지 주의사항이 있습니다. **충전 전용 케이블이 아닌 데이터 통신이 가능한 케이블**이어야 합니다 (그림 4.8 참조). 충전 전용 케이블은 전력선만 있고 데이터선이 없어서, 꽂아도 컴퓨터가 장치를 전혀 인식하지 못합니다. 스마트폰 충전용으로 쓰던 케이블이라면 되는 것도 있고 안 되는 것도 있습니다. 확실하지 않으면 Arduino나 라즈베리파이 공식 케이블을 쓰거나, 케이블 패키지에 "데이터 전송 지원"이라고 쓰인 제품을 고르세요.

> **그림 4.8 — "USB-C 케이블 — 데이터 가능 vs 충전 전용"**
>
> *촬영 지시: 케이블 두 개를 흰 배경에 나란히. 상단 케이블 위에 "데이터 OK" 라벨, 하단 케이블에 "충전 전용 X" 라벨.*

케이블이 준비됐으면, 키링의 USB-C 포트에 케이블을 꽂고 반대쪽을 노트북(또는 데스크톱) USB 포트에 연결합니다. USB-C가 딸칵 들어가는 순간, LED가 한 번 깜빡입니다. 그 깜빡임이 "나 살아있어요"라는 보드의 첫 인사입니다 (그림 4.9 참조).

> **그림 4.9 — "노트북에 처음 꽂아본 순간"**
>
> *촬영 지시: 노트북 옆 USB-C 포트에 키링이 연결된 모습. 약간 어두운 환경에서 보드 LED가 보이도록. 실제 연결 직후의 자연스러운 장면.*

이 순간 운영체제가 어떻게 반응하는지는 OS마다 다릅니다.

- **윈도우**: 처음엔 "알 수 없는 장치" 또는 "드라이버를 찾을 수 없습니다" 알림이 뜰 수 있습니다. 당황하지 마세요. 아직 드라이버를 설치하지 않았기 때문입니다. 4.5절에서 바로 해결합니다.
- **macOS**: 별도 드라이버 없이 키보드로 그냥 잡히는 경우가 많습니다 (그림 4.11 참조). "키보드 설정 보조 도구"가 뜨면 창을 닫아도 됩니다.
- **Linux**: 대부분의 배포판에서 바로 `/dev/ttyACM0` 또는 `/dev/ttyUSB0` 로 잡힙니다. `ls /dev/tty*` 로 확인하면 됩니다.

여기까지 왔으면 보드는 살아 있는 것입니다. 다음 절에서 윈도우 드라이버를 정리합니다.

## 4.5 윈도우 드라이버 처리

### 자동 설치 — Arduino IDE가 해결해 줄 때

3장에서 Arduino IDE를 이미 설치했다면, IDE가 함께 설치한 드라이버 때문에 보드를 꽂는 순간 자동으로 포트가 잡히는 경우가 많습니다. 이때는 장치 관리자(장치 관리자 단축키: `Win + X → 장치 관리자`)를 열어 **포트(COM 및 LPT)** 항목을 펼쳐 보세요. "Arduino Leonardo (COM5)" 또는 "Arduino Micro (COM3)"처럼 나타나면 성공입니다 (그림 4.13 참조). COM 번호는 컴퓨터마다 다르지만, 번호 자체는 중요하지 않습니다. Arduino IDE에서 해당 포트를 선택하면 됩니다.

> **그림 4.13 — "드라이버 설치 성공 — 'COM5' 인식"**
>
> *[스크린샷: 장치 관리자 "포트(COM 및 LPT)" 항목에 "Arduino Leonardo (COM5)"가 표시된 모습]*

### 수동 설치 — 노란 느낌표가 뜰 때

자동으로 잡히지 않으면 장치 관리자에 "기타 장치 > 알 수 없는 장치"에 노란 느낌표가 표시됩니다 (그림 4.10 참조).

> **그림 4.10 — "윈도우 장치 관리자 — '알 수 없는 장치'"**
>
> *[스크린샷: 장치 관리자 "기타 장치 > 알 수 없는 장치"에 노란 느낌표가 있는 모습]*

이 경우 수동으로 드라이버 경로를 지정해 줍니다. 순서는 다음과 같습니다.

1. 노란 느낌표 항목을 **우클릭** → **드라이버 업데이트** 선택.
2. "내 컴퓨터에서 드라이버 검색" 클릭.
3. Arduino IDE가 설치된 폴더 안의 `drivers` 하위 폴더를 지정합니다. 기본 경로는 보통 `C:\Program Files (x86)\Arduino\drivers` 또는 `C:\Users\[사용자명]\AppData\Local\Arduino15\packages\arduino\hardware\avr\...\drivers` 입니다.
4. **다음** → 드라이버가 설치되면 "성공적으로 업데이트됨" 메시지.
5. 장치 관리자 새로고침 후 COM 포트 번호 확인.

> **그림 4.12 — "드라이버 수동 설치 — .inf 경로 지정"**
>
> *[스크린샷: 장치 관리자 우클릭 → 드라이버 업데이트 → 컴퓨터에서 찾기 → Arduino 드라이버 폴더를 선택한 화면]*

### macOS / Linux는 간단합니다

macOS는 별도 드라이버가 필요 없습니다. 시스템 정보(Apple 메뉴 → 시스템 정보 → USB)를 열어 보면 키링이 "HID Keyboard" 또는 비슷한 이름으로 이미 나타납니다 (그림 4.11 참조). Linux도 마찬가지입니다. 단, Linux에서는 일반 사용자 계정이 시리얼 포트를 열 권한이 없을 수 있습니다. `sudo usermod -aG dialout $USER` 명령을 한 번 실행하고 로그아웃·재로그인하면 됩니다.

> **그림 4.11 — "macOS — 그냥 키보드로 잡힘"**
>
> *[스크린샷: macOS 시스템 정보 → USB 트리에서 키링이 HID Keyboard로 인식된 모습]*

## 4.6 더블탭 리셋 트릭과 트러블슈팅

### 왜 이걸 미리 알아야 하나

5장에서 처음으로 펌웨어를 업로드합니다. 대부분은 별 탈 없이 넘어가지만, 가끔 잘못된 코드가 올라가면 보드가 USB 장치로 인식조차 안 되는 상황이 생깁니다. 이 "먹통" 상태에서 탈출하는 유일한 방법이 **더블탭 리셋**입니다. 지금 당장 쓸 일이 없어도 손에 익혀 두면, 나중에 당황하지 않습니다.

> **그림 4.14 — "리셋 버튼 위치 확대"**
>
> *촬영 지시: PCB 위의 리셋 버튼을 매크로로 클로즈업. 손가락 끝이 버튼을 막 누르려는 자세. 흰 배경.*

### 더블탭 리셋 방법

> **그림 4.15 — "더블탭 리셋 — 8초 안에 두 번"**
>
> *[일러스트: 타임라인 형태. 첫 번째 리셋 탭 → 0~1초 안에 두 번째 탭 → "부트로더 진입 8초 창" 시작. 8초 카운트다운 시계 아이콘 포함]*

핵심은 단순합니다.

1. 리셋 버튼을 **빠르게 두 번 누릅니다** (약 0.5~1초 안에 두 번).
2. LED가 빠르게 점멸하기 시작하면 **부트로더 모드**에 진입한 것입니다.
3. 이 모드는 **8초** 동안만 유지됩니다. 8초 안에 Arduino IDE에서 업로드 버튼을 눌러야 합니다.
4. 8초가 지나면 보드가 기존 펌웨어로 부팅합니다. 실패했으면 다시 더블탭.

부트로더 모드에서는 COM 포트 번호가 바뀌는 경우가 있습니다. Arduino IDE의 포트 목록을 새로고침(Ctrl+Shift+P → "포트 선택")해서 새 포트를 선택한 뒤 업로드하세요.

### 트러블슈팅 체크리스트

보드가 전혀 인식되지 않을 때, 아래 네 가지를 순서대로 확인하세요.

> **그림 4.16 — "트러블슈팅 체크리스트"**
>
> *[일러스트/표: 네 가지 항목을 체크박스 형태로. ① 케이블이 데이터 전송 지원 케이블인가? ② USB 허브를 거치지 않고 본체 직결인가? ③ 다른 USB 포트에 꽂아봤는가? ④ 보드 LED가 켜지는가?]*

- **☐ 케이블이 데이터 지원 케이블인가?** — 충전 전용 케이블은 아무리 꽂아도 인식되지 않습니다. 케이블을 바꿔보는 것이 가장 빠른 해결책입니다.
- **☐ USB 허브를 거치지 않고 본체에 직결인가?** — 일부 저가 허브는 전류 공급이 불안정해서 보드가 제대로 초기화되지 않습니다. 모니터 뒤 허브, 키보드 사이드 허브를 모두 빼고 노트북 본체 포트에 직접 꽂아보세요.
- **☐ 다른 USB 포트에 꽂아봤는가?** — USB 포트 자체가 불량이거나 전력 출력이 낮은 경우가 있습니다. 포트를 바꿔보면 의외로 바로 해결되기도 합니다.
- **☐ 보드 LED가 켜지는가?** — LED조차 켜지지 않는다면 전원 문제입니다. 케이블과 포트를 다시 점검하거나, 혹시 보드에 물리적 손상이 없는지 눈으로 확인해 보세요.

네 가지를 모두 확인했는데도 안 된다면, 더블탭 리셋을 시도해 보세요. 리셋 직후 8초 안에 LED 점멸이 보이면 하드웨어는 정상입니다. 그 상태에서 Arduino IDE로 업로드하면 됩니다.

## 4.7 이 장을 마치며

오늘은 코드 한 줄 없이도 꽤 많은 것을 배웠습니다.

- 키링의 두뇌인 **ATmega32U4**가 어떤 칩인지 알았습니다.
- **USB HID** 덕분에 드라이버 없이도 컴퓨터가 키링을 키보드로 받아들인다는 원리를 이해했습니다.
- PCB 위 **6가지 부위**를 이름으로 구별할 수 있게 됐습니다.
- USB 케이블을 꽂아 **장치 인식**까지 확인했습니다.
- 윈도우 드라이버를 설치하고 **COM 포트 번호**를 찾았습니다.
- 비상 탈출구인 **더블탭 리셋 트릭**을 배웠습니다.

다음 5장에서는 드디어 첫 번째 펌웨어를 키링에 올립니다. 보드의 LED를 1초 간격으로 깜빡이는 "Blink" 스케치부터 시작해서, USB HID로 키 입력을 보내는 코드까지 단계별로 만들어 봅니다.
