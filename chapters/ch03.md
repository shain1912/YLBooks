## 3.1 Antigravity는 뭐고 왜 쓰는가

한 줄로 먼저 정리하겠습니다. **Claude Code는 터미널 도구, Antigravity는 IDE입니다.**

Claude Code는 검은 화면 안에서 텍스트로 대화하며 코드를 생성합니다. 파일을 직접 쓰고 읽고 명령을 실행하는 데는 뛰어나지만, 결과가 화면에 어떻게 보이는지 바로 확인하기는 어렵습니다. 게임 화면을 만들고 있는데 브라우저를 따로 열어 새로고침해야 한다면 흐름이 끊깁니다.

Antigravity는 2025년 11월 Google이 공개한 베타 IDE입니다. VS Code를 기반으로 포크(fork)하여 만들었고, Gemini 3 모델이 탑재되어 있습니다. 핵심 차이는 화면 구성입니다. 코드 에디터, 터미널, 내장 브라우저, 변경 사항 비교(diff) 뷰를 한 화면 안에 동시에 띄울 수 있습니다. 거기에 더해 여러 AI 에이전트 카드를 나란히 열어두고 각자에게 다른 작업을 맡길 수 있습니다. "왼쪽 에이전트는 HTML 레이아웃 수정, 오른쪽 에이전트는 CSS 색상 튜닝" 식의 병렬 작업이 가능합니다.

베타 기간에는 무료로 사용할 수 있습니다. 추후 가격 정책은 아직 공개되지 않았으니, 지금 써보는 것이 이득입니다.

## 3.2 가입 — 개인 Gmail 계정만

Antigravity 가입에서 학생들이 가장 많이 걸려 넘어지는 함정이 있습니다. **학교 이메일로는 로그인이 안 됩니다.**

Antigravity는 Google Workspace 계정(학교·회사에서 관리하는 `@xx대학.ac.kr`, `@회사.com` 형태)을 지원하지 않습니다. 개인 Gmail 계정(`@gmail.com`)만 허용됩니다. 학교 메일로 로그인을 시도하면 "Personal Gmail required" 안내 화면이 뜨면서 차단됩니다(그림 3.3 참조).

지금 사용하는 이메일이 학교 계정이라면 Google 계정을 새로 만들어야 합니다(그림 3.4 참조). 계정을 만드는 데 5분이면 충분합니다. `accounts.google.com`에서 '계정 만들기'를 선택하고, `@gmail.com` 주소를 새로 하나 정하면 됩니다.

개인 Gmail이 준비되었으면 Antigravity 공식 사이트에서 베타 신청을 합니다. 이메일 주소를 입력하고 신청 버튼을 누르면 대기열에 등록됩니다(그림 3.5 참조). 승인은 통상 수 일 내에 이루어지며, 승인이 완료되면 "Welcome to Antigravity beta"라는 제목의 메일이 개인 Gmail 받은편지함으로 도착합니다(그림 3.6 참조). 그 메일 안의 링크를 통해 설치 파일을 내려받을 수 있습니다.

>
> *[스크린샷: Google 계정 만들기]*

## 3.3 설치하고 첫 실행

### 윈도우

승인 메일의 링크를 클릭하면 다운로드 페이지로 이동합니다. 윈도우 사용자는 `.exe` 인스톨러 파일을 내려받습니다(그림 3.7 참조). 파일 크기는 약 200~300 MB 정도입니다. 다운로드가 완료되면 파일을 더블클릭해 설치 마법사를 실행합니다. "다음 → 다음 → 설치"를 눌러 진행하면 됩니다(그림 3.8 참조).

중요한 점이 있습니다. Antigravity는 VS Code와 **별도의 독립 프로그램**으로 설치됩니다. 기존에 VS Code를 사용하고 있어도 충돌 없이 함께 설치할 수 있습니다. 설치 경로도 다르고, 실행 파일도 다릅니다. VS Code 확장 프로그램을 다시 설치할 필요도 없습니다.

설치가 끝나면 바탕화면 또는 시작 메뉴에서 Antigravity를 찾아 실행합니다. 첫 실행 시 Gmail 로그인 창이 뜹니다(그림 3.9 참조). 앞에서 만든 개인 Gmail 계정으로 로그인하면 초기 설정이 완료됩니다.

### Mac

Mac 사용자는 같은 다운로드 페이지에서 `.dmg` 파일을 내려받아 마운트한 뒤, Antigravity 아이콘을 응용 프로그램 폴더로 드래그합니다. 이후 과정은 윈도우와 동일합니다. Apple Silicon(M1/M2/M3) 맥은 ARM 빌드가 별도로 제공되므로, 다운로드 시 "Apple Silicon" 버튼을 선택해야 합니다.

> **그림 3.7 — "윈도우 인스톨러 다운로드 페이지"**
>
> *[스크린샷: Windows .exe 다운로드 버튼 강조]*

> **그림 3.8 — "설치 진행 중 화면"**
>
> *[스크린샷: 설치 마법사]*

> **그림 3.9 — "첫 실행 — Gmail 로그인 프롬프트"**

## 3.4 Manager View 둘러보기

Antigravity를 처음 열면 VS Code와 비슷하면서도 다른 화면이 나타납니다. 이 메인 화면을 **Manager View**라고 부릅니다. 크게 네 영역으로 나뉩니다(그림 3.10 참조).

| 번호 | 위치 | 역할 |
|------|------|------|
| ① | 좌측 | 파일 트리 — 프로젝트 폴더 구조와 파일 목록 |
| ② | 중앙 | 코드 에디터 — 파일 편집 공간 |
| ③ | 우측 | 에이전트 패널 — AI와 대화하는 카드 영역 |
| ④ | 하단 | 터미널 + 내장 브라우저 탭 |

VS Code와 가장 큰 차이는 ③번 우측 에이전트 패널입니다. 여기서 AI에게 작업을 지시하면 에이전트가 직접 파일을 수정하고, 그 결과를 ④번 내장 브라우저에서 실시간으로 확인할 수 있습니다. 브라우저를 따로 열 필요가 없습니다.

또한 에이전트 카드는 여러 개를 동시에 열 수 있습니다(그림 3.11 참조). 예를 들어 카드 하나는 "게임 배경 색상 바꿔줘"를 처리하는 동안, 다른 카드는 "점수 표시 크기 키워줘"를 병렬로 처리할 수 있습니다. 처음에는 카드 한 개로 시작해도 충분합니다만, 작업이 익숙해지면 이 병렬 기능이 꽤 유용합니다.

각 카드는 자체 작업 히스토리를 갖고 있습니다. 에이전트가 파일을 수정하면 위쪽에 변경된 파일 목록이 누적되고, 클릭하면 추가/삭제된 줄이 색상으로 강조된 디프(diff) 뷰가 나타납니다. 마음에 들지 않는 변경은 한 번의 클릭으로 되돌릴 수 있습니다. AI가 만든 코드를 무비판적으로 받아들이지 않고 한 줄씩 검토하는 습관을 들이는 데 이 디프 뷰가 큰 도움이 됩니다.

자주 쓰는 단축키도 몇 가지 알아두면 작업 속도가 빨라집니다. `Ctrl + B`(맥은 `Cmd + B`)는 좌측 파일 트리를 접고 펼치며, `Ctrl + 백틱`은 하단 터미널 영역을 토글합니다. `Ctrl + Shift + P`로 명령 팔레트를 열면 모든 메뉴 항목을 검색할 수 있습니다. VS Code 단축키와 거의 동일하기 때문에, VS Code를 써본 적 있는 학생은 첫날부터 별다른 학습 곡선 없이 바로 익숙해집니다.

> **그림 3.10 — "Manager View 전체 레이아웃 (4분할)"**
>
> *[스크린샷: 메인 화면 4영역에 1~4 번호 오버레이]*

> **그림 3.11 — "에이전트 카드를 여러 개 띄운 상태"**
>
> *[스크린샷: 에이전트 카드 2~3개 동시]*

## 3.5 Claude Code와 어떻게 같이 쓸 것인가

두 도구를 모두 설치했으니, 언제 어느 도구를 쓸지 원칙을 정해야 합니다. 이 책 전체에서 일관되게 적용하는 분업 원칙은 다음과 같습니다.

> **Claude Code = 펌웨어 (.ino)  /  Antigravity = 게임 화면 (.py, .html)**

펌웨어는 PCB 키링 안에 들어가는 코드입니다. Arduino 코드(`.ino` 파일)로 작성하며, 버튼 입력 처리·LED 제어·디스플레이 드라이버 같은 내용이 담깁니다. 이 작업은 파일 수정과 명령 실행이 전부라서 Claude Code의 터미널 방식이 더 적합합니다.

게임 화면은 키링 시뮬레이터 또는 PC용 미니 게임을 만드는 코드입니다. Python(pygame), HTML/JS로 작성하며, 결과를 시각적으로 바로 확인하는 것이 중요합니다. 이때 Antigravity의 내장 브라우저와 에이전트 패널이 큰 강점을 발휘합니다.

같은 프로젝트 폴더를 양쪽에서 동시에 열어도 됩니다(그림 3.12 참조). 파일 충돌이 생기지 않도록, 한쪽에서 파일을 수정하는 동안 다른 쪽은 같은 파일을 건드리지 않으면 됩니다. 실제로 이 책의 실습에서는 Claude Code로 `.ino`를 수정하고, Antigravity로 `game.py` 또는 `index.html`을 수정하는 방식을 사용합니다.

권장 폴더 구조는 다음과 같습니다. 프로젝트 루트(예: `keyring-game/`) 안에 `firmware/` 하위 폴더를 두고 그 안에 `.ino` 파일을 모읍니다. 같은 루트의 `game/` 폴더에는 `game.py`, `chart.json`, 음원 파일 등 게임 자산을 둡니다. 이렇게 도구별로 다루는 영역이 폴더 단위로 분리되면, 한 도구가 다른 도구의 작업을 실수로 덮어쓸 가능성이 사라집니다.

루트에 `.gitignore` 파일도 하나 만들어 두세요. Antigravity와 Claude Code가 각자 만드는 캐시 폴더(`.antigravity/`, `.claude/`), 빌드 산출물(`build/`, `__pycache__/`), OS 캐시(`.DS_Store`, `Thumbs.db`)를 깃 추적에서 제외해 두면 21장에서 깃허브에 올릴 때 깔끔합니다. 처음에는 다음 한 줄만 적어도 충분합니다.

```
.antigravity/
.claude/
__pycache__/
.DS_Store
Thumbs.db
build/
```

같은 폴더를 두 도구로 열어 작업하다가 둘 다 같은 파일에 동시에 변경을 가하는 일이 생기면, 마지막에 저장한 쪽이 이깁니다(이전 변경 내용은 사라집니다). 그래서 하나의 파일은 한 번에 한 도구로만 만지는 규칙이 가장 안전합니다. 이 규칙만 지키면 두 도구를 함께 쓰는 데 별다른 문제는 없습니다.

> **그림 3.12 — "VS Code(좌) vs Antigravity(우) 인터페이스"**
>
> *[스크린샷: 같은 폴더 좌우 분할 합성. 사이드바 위치/에이전트 패널 차이]*

> **그림 3.13 — "이 책의 도구 분업 원칙"**
>
> *[일러스트: 펌웨어(.ino) → Claude Code / 게임(.py, .html) → Antigravity 화살표 분기]*

## 3.6 첫 프로젝트 — "Hello Antigravity"

설명은 잠깐 멈추고, Antigravity가 실제로 어떻게 동작하는지 한 번 직접 보겠습니다.

**1단계 — 빈 폴더 열기**

Antigravity 상단 메뉴에서 `File → Open Folder`를 선택하고, 아무 이름이나 붙인 빈 폴더를 엽니다. 예를 들어 바탕화면에 `hello-antigravity`라는 폴더를 만들어 열면 됩니다.

**2단계 — 에이전트에게 지시하기**

우측 에이전트 패널에서 새 카드를 열고 다음과 같이 입력합니다.

> 이 폴더에 Hello World HTML 파일을 만들고, 하단 브라우저로 열어줘.

에이전트가 `index.html` 파일을 생성하고, 자동으로 내장 브라우저 탭에서 결과를 보여줍니다(그림 3.14 참조).

**3단계 — 결과 확인**

하단 브라우저 영역에 "Hello, Antigravity!"라는 텍스트가 표시됩니다(그림 3.15 참조). 터미널도, 새로고침도, 외부 브라우저도 필요하지 않았습니다.

이것이 Antigravity의 핵심 경험입니다. 코드 작성·실행·시각적 확인이 한 화면 안에서 한 번에 이루어집니다.

> **그림 3.14 — "Hello World 프롬프트와 결과"**
>
> *[스크린샷: 프롬프트 입력 + index.html 생성 + 하단 브라우저 렌더링 한 화면]*

> **그림 3.15 — "내장 브라우저 프리뷰로 본 Hello World"**
>
> *[스크린샷: 하단 브라우저 영역 확대 "Hello, Antigravity!"]*
