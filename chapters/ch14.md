## 14.1 Suno 곡으로 차트 만들면 100배 재밌다

직접 고른 곡으로 리듬 게임 차트를 만들면 연습 의욕이 완전히 달라집니다. "이 곡의 이 부분에 노트를 넣어야지" 하면서 자연스럽게 박자 감각을 익힐 수 있고, 클리어했을 때의 성취감도 훨씬 큽니다.

이 책은 음원 문제를 시작부터 깔끔하게 정리했습니다. **이전 책의 강사가 [Suno](https://suno.com)에서 직접 생성한 곡들**이 본 책 깃허브 저장소(`https://github.com/ylbook/keyring-game`)의 `assets/songs/` 폴더에 들어 있습니다. 11장에서는 그중 한 곡(`song_easy.ogg`)을 기본 예제로 썼고, 14장에서는 같은 폴더의 다른 곡들 중 하나를 골라 자기만의 차트를 만들 차례입니다.

### Suno 곡을 쓰는 이유

| 항목 | 이유 |
|---|---|
| 저작권 | AI 생성물이라 K-pop·OST 같은 인접권 분쟁 위험이 없음 |
| 길이 | 리듬게임에 맞춰 1분 30초 ~ 3분 사이로 미리 잘라 둠 |
| BPM | 80~140 사이로 입문~중급 학생에게 적합한 분포 |
| 변형 | 본인이 Suno 계정만 만들면 같은 프롬프트로 변주 곡을 추가 생성 가능 |

이 장의 모든 예제는 `assets/songs/song_medium.ogg`(BPM 약 122)를 기준으로 설명합니다. 다른 곡으로 시도하려면 같은 폴더의 다른 ogg 파일 경로로 바꾸기만 하면 됩니다.

> **그림 14.2 — "Suno 곡 폴더 구성 — 입문/중급/고급"**
> *[도식: 폴더 트리 / `assets/songs/` 안에 `song_easy.ogg`(BPM 90) / `song_medium.ogg`(BPM 122) / `song_hard.ogg`(BPM 140) 세 항목 / 각 파일 옆에 짧은 설명 배지]*

### 본인이 직접 Suno로 만들고 싶다면

학생이 직접 Suno 계정을 만들어 곡을 생성하고 싶다면 `https://suno.com`에서 무료 크레딧으로 시작할 수 있습니다. 프롬프트 작성 팁:

- 장르·BPM·분위기를 명확히: "upbeat chiptune, 120 BPM, 90 seconds, simple drum pattern"
- 가사가 없는 인스트루멘털을 권장(차트 만들 때 박자 잡기 쉬움)
- 생성 후 mp3로 받아지면 Audacity로 ogg 변환

다만 본 책 본문은 강사가 미리 만든 곡을 기준으로 진행합니다. 자기 생성 곡은 이 장 마지막 도전 과제로 시도해 보세요.

## 14.2 BPM 측정 — 탭 템포 방식

차트를 만들려면 먼저 곡의 **BPM(Beats Per Minute)**을 알아야 합니다. BPM이 정확해야 노트가 박자에 딱딱 맞아 떨어집니다.

BPM을 측정하는 가장 직관적인 방법은 **탭 템포(Tap Tempo)**입니다. 스마트폰 박자 기계 앱이나 DJ 소프트웨어도 탭 템포 방식을 표준으로 씁니다. 원리는 단순합니다. 곡을 들으면서 박자에 맞춰 버튼을 두드리면, 두드린 시각 사이의 평균 간격을 구해 BPM을 계산합니다.

예를 들어 스페이스바를 두드린 시각이 아래처럼 기록됐다고 합시다(단위: 초).

```
0.00  0.49  0.98  1.47  1.96  2.45  2.94  3.43
```

연속된 두 값의 차이(간격)는 모두 0.49초에 가깝습니다. 이 평균 간격을 `t`초라 하면 BPM은 다음과 같이 계산됩니다.

```
BPM = 60 / t = 60 / 0.49 ≈ 122.4
```

> **그림 14.3 — "탭 템포 원리 — 두드린 간격의 평균"**
> *[도식: 수평 시간축 위에 8개 점 / 점 사이마다 양방향 화살표로 간격 표시 / 화살표 아래 "평균 → BPM" 레이블]*

### 몇 번 두드려야 정확한가?

탭 횟수가 많을수록 평균이 안정됩니다. 8회 입력이면 오차가 ±3 BPM 수준이고, 16회 입력이면 ±1 BPM까지 줄어듭니다 (그림 14.6 참조). 빠른 템포 확인이 필요하면 8회로 충분하지만, 정밀한 차트 작업을 할 때는 16회를 권장합니다.

## 14.3 탭 템포 도구 직접 만들기

원리를 알았으니 직접 도구를 만들어 봅시다. Antigravity에 다음과 같이 부탁합니다.

> **그림 14.4 — "Antigravity에 탭 템포 도구 부탁"**
> *[스크린샷: Antigravity 입력창 / 프롬프트 "스페이스바를 여러 번 두드리면 두드린 간격의 평균으로 BPM을 계산하는 파이썬 스크립트를 30줄 이내로 만들어줘"]*

```
스페이스바를 여러 번 두드리면 두드린 간격의 평균으로 BPM을 계산하는
파이썬 스크립트를 30줄 이내로 만들어줘.
Enter 키를 누르면 입력을 끝내고 결과를 출력하는 방식으로.
```

Antigravity가 생성해 줄 코드는 대략 아래와 같습니다.

```python
# tap_tempo.py
import time

def tap_tempo():
    print("스페이스바를 박자에 맞춰 두드리세요. (Enter로 종료)")
    timestamps = []
    while True:
        key = input()
        if key == "":           # Enter 입력 → 종료
            break
        timestamps.append(time.time())

    if len(timestamps) < 2:
        print("두 번 이상 입력해야 BPM을 계산할 수 있습니다.")
        return

    intervals = [
        timestamps[i+1] - timestamps[i]
        for i in range(len(timestamps) - 1)
    ]
    avg_interval = sum(intervals) / len(intervals)
    bpm = 60 / avg_interval
    count = len(timestamps)
    print(f"{count}회 입력 평균: {bpm:.1f} BPM")

if __name__ == "__main__":
    tap_tempo()
```

터미널에서 실행하면 아래처럼 동작합니다. 스페이스바를 누를 때마다 엔터를 대신 입력한다고 생각하면 됩니다. 16번 두드린 뒤 Enter를 누르면 결과가 나옵니다.

> **그림 14.5 — "탭 템포 스크립트 실행 결과"**
> *[스크린샷: 터미널 / "스페이스바를 박자에 맞춰 두드리세요." 출력 후 여러 줄 공백 입력 / 마지막 줄 "16회 입력 평균: 122.4 BPM"]*

> **그림 14.6 — "BPM 추정 정확도 — 8회 vs 16회"**
> *[도식: 세로축 오차(±BPM) / 가로축 입력 횟수 / 막대 두 개: 8회 → ±3 BPM, 16회 → ±1 BPM]*

AI의 강점은 "이 정도면 충분히 쓸 수 있는 스크립트"를 30줄 안에 즉시 만들어 준다는 점입니다. 여러분은 원리만 이해하고 있으면 됩니다. 코드를 처음부터 외울 필요는 없습니다.

측정한 BPM 값은 메모해 두세요. 이어지는 절에서 차트 에디터에 입력하게 됩니다.

## 14.4 노트 배치 가이드 — 좋은 차트 vs 나쁜 차트

BPM을 알았다고 해서 노트를 아무 위치에나 넣으면 차트가 어색해집니다. 좋은 차트에는 미학이 있습니다.

### 박자 강세에 노트 맞추기

음악에는 **강박(강세가 있는 박자)**과 **약박**이 있습니다. 4/4박자 기준으로 1·3박이 강박, 2·4박이 약박입니다. 좋은 리듬 게임 차트는 강박에 굵은 노트, 약박에 가벼운 노트를 배치해 음악의 리듬을 손으로 느끼게 해 줍니다.

### 4비트 · 8비트 · 16비트 패턴

- **4비트** — 한 마디에 4개 노트. 초보자 구간에 적합.
- **8비트** — 한 마디에 8개 노트. 부드러운 멜로디 표현.
- **16비트** — 한 마디에 16개 노트. 빠른 드럼 필인, 숙련자 구간.

> **그림 14.7 — "노트 배치 패턴 — 4비트·8비트·16비트"**
> *[도식: 수평 시간축 3줄 / 위쪽 줄 점 4개(4비트) / 가운데 줄 점 8개(8비트) / 아래쪽 줄 점 16개(16비트) / 한 마디 길이 동일]*

### 좋은 차트 vs 나쁜 차트

| 좋은 차트 | 나쁜 차트 |
|---|---|
| 강박에 노트 집중, 약박에 여백 | 무작위 위치에 노트 |
| 레인을 골고루 활용 | 같은 레인 연속 8개 이상 |
| 코러스 직전 잠깐 쉬어가기 | 처음부터 끝까지 16비트 |
| 핵심 멜로디 라인을 따라가기 | 드럼 박자만 계속 반복 |

특히 **같은 레인 연타**는 주의해야 합니다. 사람의 손가락은 같은 키를 초당 10회 이상 빠르게 반복하는 데 한계가 있습니다. 16비트 연타를 같은 레인에 배치하면 클리어가 거의 불가능해집니다. 같은 박자 패턴이라도 레인을 번갈아 가며 분산시키면 훨씬 손에 잘 맞습니다 (그림 14.8 참조).

> **그림 14.8 — "좋은 차트 vs 나쁜 차트"**
> *[도식: 왼쪽 O — 강세 따라간 차트(레인 분산, 강박 강조) / 오른쪽 X — 같은 레인 16연타 차트(빨간 X 표시)]*

## 14.5 간단 차트 에디터 만들기

이제 노트를 마우스로 직접 클릭해서 배치할 수 있는 차트 에디터를 만들 차례입니다. 이것은 지금까지 만든 것 중 가장 복잡한 도구입니다. 하지만 Antigravity가 곁에 있으면 충분히 해낼 수 있습니다.

Antigravity에 다음 사양을 설명하세요.

> **그림 14.9 — "Antigravity로 차트 에디터 GUI 만들기"**
> *[스크린샷: Antigravity 입력창 / 에디터 사양 프롬프트 전문 입력 중인 화면]*

```
pygame으로 간단한 리듬 게임 차트 에디터를 만들어줘.
- 창 크기 800x600
- 세로 시간축, 가로로 4개 레인
- mp3 파일을 재생하면서 마우스 클릭으로 노트를 추가할 수 있어야 해
- 클릭한 위치(레인 번호, 시간)를 리스트에 저장
- S 키를 누르면 notes 리스트를 chart.json으로 저장

```

Antigravity가 생성한 코드를 `chart_editor.py`로 저장하면 아래와 같은 화면이 나타납니다.

> **그림 14.10 — "차트 에디터 화면 — 음악 재생 + 4레인 + 클릭으로 노트 찍기"**
> *[스크린샷: 에디터 창 / 회색 배경 + 세로 구분선 4레인 / 흰 점들이 찍혀있음 / 현재 재생 위치 가로 빨간 막대]*

에디터 화면에서 마우스를 클릭하면 해당 위치에 노트가 생성됩니다. 음악이 재생되면서 현재 위치를 나타내는 가로 막대가 위에서 아래로 내려옵니다. 곡을 들으며 클릭하면 자연스럽게 박자에 맞는 노트를 배치할 수 있습니다.

작업이 끝나면 `S` 키를 눌러 저장합니다. `chart.json` 파일이 생성됩니다.

> **그림 14.11 — "찍은 노트가 chart.json으로 저장"**
> *[스크린샷 좌우: 왼쪽 — 에디터 화면 노트 배치 완료 / 오른쪽 — 저장된 chart.json 일부 (레인 번호와 시간 값 목록)]*

저장된 JSON 파일의 구조는 대략 이렇습니다.

```json
{
  "bpm": 122.4,
  "song": "assets/songs/song_medium.ogg",
  "notes": [
    {"lane": 0, "time": 0.49},
    {"lane": 2, "time": 0.98},
    {"lane": 1, "time": 1.47},
    {"lane": 3, "time": 1.96}
  ]
}
```

`lane`은 0~3 사이의 레인 번호, `time`은 곡 시작부터의 경과 시간(초)입니다. 이 형식은 13장에서 만든 게임 엔진이 그대로 읽을 수 있습니다.

## 14.6 자기 차트로 플레이 + 트러블슈팅

`chart.json`을 저장했으면 실제 게임에서 로드해 플레이해 봅시다. 13장에서 만든 게임 코드에서 차트 파일 경로만 `chart.json`으로 바꾸면 됩니다.

> **그림 14.12 — "자기 차트로 첫 플레이 (학생 손)"**
> *[촬영 사진: 흰 책상 위 PCB 키링 + 학생 손 올린 모습 + 모니터에 자기가 만든 차트 플레이 화면 / 자연광]*

처음 플레이해 보면 두 가지 문제가 자주 생깁니다.

### 문제 1 — 노트가 너무 많을 때

16비트 패턴을 너무 많이 넣으면 손이 따라가지 못합니다. 이럴 때는 에디터에서 16비트 노트를 절반만 남기는 **솎아내기** 작업을 합니다. 두 개 중 약박에 해당하는 노트를 지우면 8비트 패턴이 됩니다.

> **그림 14.13 — "노트 너무 많을 때 — 16비트를 8비트로 솎아내기"**
> *[도식: 왼쪽 — 빽빽한 16비트 차트 / 화살표 / 오른쪽 — 약박 노트 제거한 8비트 차트]*

솎아내는 기준은 간단합니다. 16비트 노트 중 홀수 번째(1, 3, 5…번째)만 남기면 8비트가 됩니다. `chart.json`을 직접 편집하거나, 에디터에서 해당 노트를 우클릭으로 삭제하도록 Antigravity에 기능 추가를 부탁해도 됩니다.

### 문제 2 — BPM이 빗나갔을 때

곡 초반에는 노트가 박자에 맞는데, 후반으로 갈수록 점점 어긋나는 현상이 생길 수 있습니다. 원인은 측정한 BPM이 실제 곡의 BPM과 조금 다르기 때문입니다. 0.1 BPM 차이도 1분짜리 곡의 끝에서는 수십 개 노트가 밀릴 수 있습니다.

> **그림 14.14 — "BPM이 빗나갔을 때 — 곡 후반에서 노트가 어긋나는 증상"**
> *[스크린샷: pygame / 곡 후반 레인 노트가 박자보다 한 박 늦게 내려오고 있음 / 빨간 화살표로 "어긋남" 메모]*

해결 방법은 두 가지입니다.

1. **탭 템포를 더 많이 입력한다** — 16회 대신 32회까지 늘려 평균을 정밀하게 만듭니다.
2. **오디오 편집기로 BPM을 확인한다** — Audacity 같은 무료 프로그램에서 파형을 보고 BPM을 직접 읽을 수 있습니다.

보통 ±0.5 BPM 오차라면 3분짜리 곡 기준으로 후반 노트가 약 1박 정도 밀립니다. `chart.json`의 `"bpm"` 값을 0.1씩 조정해 가며 플레이해 보면 금방 찾을 수 있습니다.

## 14.7 다음 장 예고 — 발표 준비

수고했습니다! 이 장에서 우리가 한 것을 정리하면 다음과 같습니다.
