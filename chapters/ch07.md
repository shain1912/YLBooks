# 7장 채터링과의 전쟁 — 디바운싱

---

## 7.1 한 번 눌렀는데 'aaaaa' — 채터링 현상

6장에서 키링의 첫 번째 버튼이 드디어 'a'를 입력하는 순간, 분명히 쾌감이 있었을 것입니다. 그런데 메모장을 열어놓고 버튼을 빠르게 몇 번 눌러보면 어느 순간 황당한 장면이 펼쳐집니다. 한 번 누른 것 같은데 화면에는 `aaaaa` — 다섯 글자가 찍혀 있습니다. 심하면 `aaaaaaaaaa`처럼 열 글자가 한꺼번에 나타나기도 합니다.

> **그림 7.1 — "디바운싱 전: 한 번 눌렀는데 'aaaaa' 입력"**
>
> *촬영 지시: 메모장에 'aaaaa' 5~10개 + 손가락이 막 첫 번째 버튼에서 떨어진 자세. 위에서 내려다보는 각도.*

분명히 한 번만 눌렀습니다. 그런데 왜 여러 글자가 입력될까요?

원인은 기계식 스위치의 물리적인 특성에 있습니다. 우리가 버튼을 누르는 동작은 사람 입장에서는 순식간이지만, 스위치 내부의 금속 접점 두 개가 서로 닿는 과정을 마이크로초 단위로 들여다보면 전혀 다른 이야기가 펼쳐집니다. 접점이 처음 닿는 순간 완전히 붙지 못하고 1~5밀리초 동안 수십 번 미세하게 붙었다 떨어졌다를 반복합니다. 이 현상을 **채터링(chattering)**, 혹은 **바운싱(bouncing)**이라고 합니다.

Arduino의 `digitalRead()`는 이 1밀리초짜리 떨림을 그대로 읽어들입니다. 마이크로컨트롤러의 루프 속도는 사람이 체감하는 속도보다 수백 배 빠르기 때문에, 접점이 떨리는 짧은 순간에도 LOW 신호를 여러 번 감지해 'a'를 여러 번 입력하는 것입니다.

> **그림 7.2 — "기계식 스위치 채터링 파형 (개념도)"**
>
> [일러스트: 시간 vs 전압 그래프. X축은 시간(ms), Y축은 전압(V). 버튼을 누르는 순간 1~5ms 구간에서 HIGH/LOW가 불규칙하게 진동하다 안정화. 디지털 임계값 라인(점선)을 파형이 여러 번 교차하는 것을 표시.]

채터링은 불량 스위치에서만 생기는 현상이 아닙니다. 체리(Cherry), 카일(Kailh) 같은 고급 기계식 스위치도 모두 채터링이 있습니다. 이것은 어쩔 수 없는 기계적 특성이기 때문에, **소프트웨어에서 걸러내는 것**이 올바른 해결책입니다. 그 기법을 디바운싱(debouncing)이라고 합니다.

이 챕터 전체가 바로 그 디바운싱을 제대로 구현하는 과정입니다. 끝에 가면 버튼을 아무리 빠르게 눌러도 정확히 한 글자씩만 입력되는 코드를 갖게 됩니다.

## 7.2 오실로스코프 없이 채터링 시각화하기

채터링이 실제로 일어나고 있다는 것을 어떻게 확인할 수 있을까요? 전자공학과 실험실에는 오실로스코프라는 장비가 있어서 신호의 떨림을 파형으로 보여줍니다. 하지만 우리에게는 그런 장비가 없어도 됩니다. Arduino IDE 안에 내장된 두 가지 도구로 충분합니다.

먼저 시리얼 모니터에서 날 것의 `digitalRead` 값을 빠르게 출력해 보겠습니다. 아래 코드를 Arduino IDE에 붙여 넣고 PCB의 첫 번째 버튼(이미 연결되어 있는 버튼)에 대해 업로드합니다.

```cpp
// 7장 — 채터링 관찰용 코드 (업로드 후 시리얼 모니터 확인)
const int BUTTON_PIN = 2;  // PCB에 이미 배선된 첫 번째 버튼 핀

void setup() {
  Serial.begin(115200);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
}

void loop() {
  int val = digitalRead(BUTTON_PIN);
  Serial.println(val);
  // delay 없이 최대한 빠르게 읽는다
}
```

업로드 후 시리얼 모니터를 열고 우측 하단 보드레이트를 **115200**으로 맞춥니다. 버튼을 누르지 않은 상태에서는 `1`이 빠르게 흘러내려옵니다. 이제 버튼을 살짝 눌러보세요. 눌리는 순간 `1 0 1 0 0 1 0 0 0 1 1 0 0 ...`처럼 0과 1이 뒤섞여서 흘러나오는 구간이 보입니다. 그게 바로 채터링입니다 (그림 7.3 참조).

> **그림 7.3 — "시리얼 모니터로 본 raw digitalRead 값"**
>
> [스크린샷: 시리얼 모니터에 `1/0/0/1/0/0/0/1/1/0...` 빠른 변화가 흘러내려오는 장면. 0과 1이 뒤섞이는 구간에 빨간 박스 표시.]

이번에는 같은 코드로 시리얼 **플로터**를 사용해보겠습니다. 시리얼 플로터는 `Serial.println()` 값을 실시간 그래프로 그려주는 도구로, Arduino IDE에 기본으로 내장되어 있습니다. 잘 알려지지 않아서 처음 보는 분도 많을 텐데, 메뉴 위치는 다음과 같습니다.

**Arduino IDE 메뉴 → 도구(Tools) → 시리얼 플로터(Serial Plotter)**

단축키는 `Ctrl + Shift + L`입니다. 시리얼 플로터를 열고 버튼을 누르면 그래프가 누르는 순간 들쭉날쭉해지는 모습이 실시간으로 그려집니다 (그림 7.4 참조). 오실로스코프 없이도 채터링 파형을 눈으로 확인할 수 있습니다.

> **그림 7.4 — "시리얼 플로터 — 채터링 그래프"**
>
> [스크린샷: 시리얼 플로터에 그려진 그래프. 대부분 HIGH(1) 수평선이다가 버튼 누르는 순간 LOW/HIGH가 격렬하게 진동. 해당 구간에 빨간 박스 표시.]

이제 문제가 눈에 보입니다. 다음 단계는 이것을 코드에서 걸러내는 것입니다.

## 7.3 디바운싱 두 가지 접근

채터링을 소프트웨어로 걸러내는 방법에는 크게 두 가지가 있습니다.

**첫 번째: `millis()`로 직접 시간 체크**

마지막으로 버튼 상태가 바뀐 시각을 기록해두고, 그 이후 일정 시간(예: 10ms)이 지나야만 새로운 입력으로 인정하는 방식입니다. 원리가 명확해서 디바운싱이 무엇인지 이해하기에 좋습니다.

```cpp
// millis() 직접 구현 — 학습용
const int BUTTON_PIN = 2;
const unsigned long DEBOUNCE_DELAY = 10; // ms

int lastButtonState = HIGH;
int buttonState = HIGH;
unsigned long lastDebounceTime = 0;

void loop() {
  int reading = digitalRead(BUTTON_PIN);

  if (reading != lastButtonState) {
    lastDebounceTime = millis();
  }

  if ((millis() - lastDebounceTime) > DEBOUNCE_DELAY) {
    if (reading != buttonState) {
      buttonState = reading;
      if (buttonState == LOW) {
        // 버튼 눌림 처리
      }
    }
  }
  lastButtonState = reading;
}
```

원리는 간단하지만 코드가 꽤 길어지고, 버튼이 4개가 되면 같은 로직을 4번 반복해야 합니다. 실수도 늘어납니다.

**두 번째: Bounce2 라이브러리**

커뮤니티에서 검증된 디바운싱 라이브러리를 사용하는 방식입니다. 위의 복잡한 시간 비교 로직을 내부에서 처리해주기 때문에 코드가 훨씬 간결해집니다. 버튼이 4개여도 객체 4개만 만들면 됩니다.

> **그림 7.5 — "디바운싱 두 접근 비교"**
>
> [표: 비교 항목 / millis() 직접 구현 / Bounce2 라이브러리
> - 구현 코드 줄 수: ~20줄 / ~8줄
> - 가독성: 중간 (조건 중첩) / 높음 (메서드 이름 직관적)
> - 버튼 4개 시: 반복 코드 4세트 / 객체 4개
> - 학습 가치: 원리 이해에 좋음 / 실무 패턴 습득
> - 본 책 채택: — / ✓]

원리를 이해하는 데는 `millis()` 직접 구현이 유익합니다. 하지만 실제 게임기 펌웨어를 만들 때는 유지보수가 쉽고 검증된 Bounce2를 쓰는 것이 훨씬 현명합니다. **본 책은 7장부터 Bounce2 라이브러리를 채택합니다.** 버튼이 늘어나도, 로직이 복잡해져도 코드가 깔끔하게 유지됩니다.

## 7.4 Bounce2 라이브러리 설치

Bounce2는 Thomas Fredericks가 만들고 수년간 커뮤니티에서 검증된 오픈소스 라이브러리입니다. Arduino IDE의 라이브러리 매니저에서 바로 설치할 수 있습니다.

**설치 순서:**

1. Arduino IDE를 열고 메뉴에서 **스케치(Sketch) → 라이브러리 포함하기(Include Library) → 라이브러리 관리...(Manage Libraries...)** 를 선택합니다. 단축키는 `Ctrl + Shift + I`입니다.

2. 라이브러리 매니저 검색창에 `Bounce2`를 입력합니다.

3. 검색 결과에서 **"Bounce2" by Thomas Fredericks**를 찾아 **[INSTALL]** 버튼을 클릭합니다.

> **그림 7.6 — "라이브러리 매니저에서 Bounce2 검색"**
>
> [스크린샷: 라이브러리 매니저 창. 검색창에 "Bounce2" 입력. 결과 목록에 "Bounce2 by Thomas Fredericks" 항목이 보이고 [INSTALL] 버튼 강조 표시.]

4. 설치가 완료되면 버전 번호 옆에 **"INSTALLED"** 라벨이 나타납니다. 설치 완료 메시지가 하단 콘솔에도 출력됩니다.

> **그림 7.7 — "Bounce2 설치 완료 메시지"**
>
> [스크린샷: 라이브러리 매니저에서 Bounce2 항목 옆 "INSTALLED" 녹색 라벨. 하단 출력 콘솔에 설치 완료 메시지.]

설치 후 예제 코드를 확인하고 싶다면 **파일(File) → 예제(Examples) → Bounce2** 메뉴에서 기본 예제를 바로 열 수 있습니다. 시간이 있을 때 한번 훑어보면 라이브러리 구조를 이해하는 데 도움이 됩니다.

## 7.5 fell()과 rose() — 눌리는 순간, 떼는 순간만 잡기

Bounce2의 핵심은 `fell()`과 `rose()` 두 메서드입니다. 이름부터 직관적입니다.

- **fell()** — 신호가 HIGH에서 LOW로 *떨어진* 순간, 즉 버튼이 눌린 바로 그 순간 한 번만 `true`를 반환합니다.
- **rose()** — 신호가 LOW에서 HIGH로 *올라온* 순간, 즉 버튼이 떼어진 바로 그 순간 한 번만 `true`를 반환합니다.

> **그림 7.8 — "fell() / rose() 동작 시점"**
>
> [일러스트: 시간 축 위에 버튼 신호 파형(디바운싱 처리 후 깔끔한 구형파). 신호가 HIGH→LOW로 떨어지는 에지에 아래 방향 화살표 + "fell() → true (1 tick)" 레이블. 신호가 LOW→HIGH로 올라오는 에지에 위 방향 화살표 + "rose() → true (1 tick)" 레이블. 그 외 구간에서는 false 표시.]

평소에는 두 메서드 모두 `false`를 반환하다가, 신호 변화가 있는 딱 그 루프 사이클 한 번만 `true`가 됩니다. 채터링으로 인한 수십 번의 ON/OFF 진동이 있더라도, 디바운스 시간이 지나 안정된 상태가 확인된 다음에야 `fell()` 또는 `rose()`가 `true`를 반환합니다. 덕분에 "눌림"과 "떼임"이 정확히 한 번씩만 감지됩니다.

이 동작 방식은 비유로 이해하면 쉽습니다. 친구가 문을 두드리는 상황을 상상해보세요. 노크 소리가 여러 번 울려도 "문이 열렸다"는 판단은 도어락 센서가 완전히 닫힘 상태에서 열림 상태로 바뀐 그 순간, 딱 한 번만 기록합니다. `fell()`과 `rose()`가 하는 일이 정확히 그것입니다.

**디바운스 시간 설정**

Bounce2에서는 `interval()` 메서드로 디바운스 대기 시간을 밀리초 단위로 설정합니다. 이 값은 얼마로 해야 할까요?

> **그림 7.9 — "디바운스 시간 5ms / 10ms / 50ms 비교"**
>
> [표: 디바운스 시간 / 반응 속도 / 채터링 억제 / 게임 적합성
> - 5ms: 매우 빠름 / 약함 (일부 채터링 통과 가능) / 주의 필요
> - 10ms: 빠름 / 강함 / 권장 (게임용)
> - 50ms: 느림 / 매우 강함 / 연타 인식 어려움]

대부분의 기계식 스위치는 채터링이 5ms 이내에 끝납니다. 따라서 10ms면 채터링은 완전히 걸러지면서도, 초당 100번 입력까지 인식할 수 있는 충분한 반응 속도가 나옵니다. **게임용으로는 10ms가 표준 권장값입니다.** 본 책도 이 값을 사용합니다.

기본 사용 패턴은 다음과 같습니다.

```cpp
#include <Bounce2.h>

Bounce bouncer = Bounce();   // Bounce2 객체 생성

void setup() {
  bouncer.attach(2, INPUT_PULLUP); // PCB 첫 번째 버튼 핀 연결
  bouncer.interval(10);            // 디바운스 시간 10ms 설정
}

void loop() {
  bouncer.update();                // 반드시 매 루프마다 호출

  if (bouncer.fell()) {
    // 버튼이 눌린 순간 — 여기에 동작 삽입
  }
  if (bouncer.rose()) {
    // 버튼이 떼어진 순간 — 여기에 동작 삽입
  }
}
```

`update()`를 `loop()` 안에서 빠뜨리면 Bounce2가 신호를 감지하지 못합니다. 매 루프마다 반드시 호출해야 한다는 점을 기억해두세요.

## 7.6 Claude Code로 1버튼 코드 리팩토링하기

이제 6장에서 완성한 1버튼 키보드 코드를 Bounce2 기반으로 다시 작성할 차례입니다. 직접 코드를 고쳐도 되지만, 바이브코딩 방식대로 Claude Code에게 부탁해보겠습니다.

먼저 6장 코드를 다시 봅니다. 아마 이런 구조였을 것입니다.

```cpp
// 6장 원본 코드 — 디바운싱 없음
#include <Keyboard.h>

const int BUTTON_PIN = 2;
int lastButtonState = HIGH;

void setup() {
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  Keyboard.begin();
}

void loop() {
  int buttonState = digitalRead(BUTTON_PIN);

  if (buttonState == LOW && lastButtonState == HIGH) {
    Keyboard.press('a');
  }
  if (buttonState == HIGH && lastButtonState == LOW) {
    Keyboard.release('a');
  }

  lastButtonState = buttonState;
}
```

이 코드에는 디바운싱이 전혀 없습니다. `digitalRead`의 날 것 값을 그대로 쓰기 때문에 채터링이 있으면 'a'가 여러 번 입력됩니다.

**Claude Code 프롬프트 예시**

터미널에서 Claude Code를 실행하고, 6장 .ino 파일이 있는 디렉터리로 이동한 다음 이렇게 요청합니다.

> **그림 7.10 — "Claude Code 리팩토링 프롬프트"**
>
> [스크린샷: 터미널에 다음 프롬프트 입력. "아래 Arduino 코드를 Bounce2 라이브러리 기반으로 리팩토링해줘. 조건: 1) Bounce2 attach/interval/update/fell/rose 사용, 2) 디바운스 10ms, 3) fell()일 때 Keyboard.press, rose()일 때 Keyboard.release, 4) 주석 한국어. 코드만 출력해줘."]

Claude Code가 리팩토링한 결과 코드는 다음과 같습니다.

```cpp
// 7장 — Bounce2 적용 1버튼 키보드
#include <Keyboard.h>
#include <Bounce2.h>

const int BUTTON_PIN = 2;  // PCB에 이미 배선된 첫 번째 버튼

Bounce bouncer = Bounce();  // Bounce2 디바운서 객체

void setup() {
  Keyboard.begin();
  bouncer.attach(BUTTON_PIN, INPUT_PULLUP);  // 핀 연결 + 풀업 설정
  bouncer.interval(10);                       // 디바운스 10ms
}

void loop() {
  bouncer.update();  // 매 루프마다 신호 상태 갱신 — 필수

  if (bouncer.fell()) {
    // 버튼이 눌린 순간 (HIGH → LOW) 딱 한 번만 실행
    Keyboard.press('a');
  }

  if (bouncer.rose()) {
    // 버튼이 떼어진 순간 (LOW → HIGH) 딱 한 번만 실행
    Keyboard.release('a');
  }
}
```

> **그림 7.11 — "Before / After 코드 비교"**
>
> [스크린샷: 좌측 6장 원본 코드 / 우측 7장 Bounce2 적용 코드. 삭제된 줄은 빨간 배경, 추가된 줄은 초록 배경. `#include <Bounce2.h>`, `Bounce bouncer`, `bouncer.attach`, `bouncer.interval`, `bouncer.update()`, `bouncer.fell()`, `bouncer.rose()` 줄 강조.]

6장 코드와 비교하면 줄 수 차이는 크지 않지만, `digitalRead`를 직접 비교하는 로직이 완전히 사라지고 Bounce2 메서드로 대체된 것을 볼 수 있습니다.

> **그림 7.12 — "Bounce2 핵심 4줄 설명"**
>
> [일러스트: 코드 스니펫 배경 위에 4개 화살표. `bouncer.attach(핀, INPUT_PULLUP)` → "이 핀의 신호를 모니터링 시작". `bouncer.interval(10)` → "10ms 안의 떨림은 무시". `bouncer.update()` → "현재 상태 갱신 (매 루프 필수)". `bouncer.fell()` → "눌린 순간만 true 반환".]

코드를 이해할 때는 이 네 줄의 역할을 중심으로 읽으면 전체 흐름이 보입니다.

## 7.7 결과 — 한 번 눌러 한 글자만

Bounce2 코드를 Arduino IDE에서 컴파일하고 키링에 업로드합니다. 업로드가 완료되면 메모장을 열고 버튼을 눌러봅니다.

이제 한 번 누르면 정확히 'a' 한 글자만 입력됩니다 (그림 7.13 참조).

> **그림 7.13 — "디바운싱 후: 정확히 한 글자만"**
>
> *촬영 지시: 메모장에 'a' 한 글자만 입력된 화면 + 버튼에서 손가락이 막 떼어지는 자세. 위에서.*

더 확실하게 확인하려면 연타 테스트를 해봅니다. 버튼을 최대한 빠르게 10번 눌러보세요. 메모장에 `aaaaaaaaaa` — 정확히 10개의 'a'가 찍힙니다. 6장에서는 손가락 한 번 움직임에 5~10개씩 쏟아지던 글자가, 이제는 내 의지대로 정확히 한 번씩만 찍힙니다 (그림 7.14 참조).

> **그림 7.14 — "빠르게 연타해도 한 번에 한 글자"**
>
> *촬영 지시: 메모장에 'aaaaaaaaaa' 10글자 균일하게 입력된 화면 + 손가락 연타 흐릿하게(셔터 속도 1/30초 이하). 위에서.*

이 정밀도가 게임에서 왜 중요한지 생각해보세요. OSU!같은 리듬 게임에서 버튼 하나의 입력이 두 번으로 인식되면 콤보가 끊어지고 점수가 떨어집니다. 타이핑 연습 게임이라면 자기도 모르게 오타가 늘어납니다. 키링을 단순한 USB 장치가 아니라 **믿을 수 있는 입력 도구**로 만들려면 디바운싱은 필수입니다.

이제 4개의 버튼 모두에 디바운싱을 적용하는 준비가 되었습니다. 다음 장에서는 Bounce2 객체를 4개로 늘리고, 각 버튼에 다른 키를 매핑해서 실제로 OSU!를 플레이할 수 있는 완성 펌웨어를 만들 것입니다. 7장에서 익힌 `fell()`/`rose()` 패턴이 그대로 4배로 확장됩니다.

---

**7장 핵심 정리**

- 기계식 스위치 접점이 닫힐 때 1~5ms 동안 ON/OFF를 반복하는 현상을 채터링이라 한다.
- 채터링은 Arduino `digitalRead()`로 그대로 읽히기 때문에 소프트웨어 디바운싱으로 걸러야 한다.
- Arduino IDE 시리얼 플로터(**도구 → 시리얼 플로터**)로 채터링 파형을 시각화할 수 있다.
- Bounce2 라이브러리(`attach` → `interval` → `update` → `fell`/`rose`)로 디바운싱을 4줄로 구현한다.
- 디바운스 시간은 게임용 기준 **10ms**가 표준 권장값이다.
- `fell()`은 눌린 순간, `rose()`는 떼어진 순간 딱 한 번만 `true`를 반환한다.
