# 촬영·일러스트 지시서 일람

이 책의 모든 그림 자리표시자 캡션과 지시문을 한 파일에 모았습니다.
사진 알바·일러스트레이터에게 이 파일 한 장을 그대로 넘기면 됩니다.

## 카테고리 구분
- **촬영**: 알바 사진사가 자연광 + 데스크 램프 + 흰 종이 배경 기준으로 찍는 실사
- **스크린샷**: 저자가 본인 노트북에서 직접 캡처 (알바 발주 X)
- **일러스트**: 일러스트레이터 또는 작가가 그리는 도식·다이어그램·손그림
- **표**: 마크다운 표로 본문에서 직접 처리
- **기타**: 위 분류 외 — 합성·시퀀스·페이지 미리보기 등 (수동 검토 필요)

## 통계

- 총 그림: **350장**
- 촬영 (알바 발주): **62장**
- 스크린샷 (저자 직접): **201장**

## 4장 키링 보드 처음 만나기

- **그림 4.1** `[촬영]` — "PCB 키링 보드 — 동전과 크기 비교"
  - 흰 A4 용지 위 top-down 앵글. 키링 PCB 보드와 500원 동전을 나란히 배치. 자연광 또는 데스크 램프. 그림자 최소화.
- **그림 4.2** `[촬영]` — "ATmega32U4 칩 클로즈업"
  - PCB 메인 칩을 매크로 렌즈로 클로즈업. "ATMEGA32U4" 각인이 뚜렷이 보이도록 핀트 맞춤. 흰 배경.
- **그림 4.4** `[표]` — "주요 사양 표"
  - 칩 ATmega32U4 / 플래시 32KB / SRAM 2.5KB / 클럭 16MHz / USB Full Speed / 가격 1.5~2만원
- **그림 4.3** `[일러스트]` — "USB HID 개념도"
  - 키링 → USB 케이블 → 컴퓨터 순서로 화살표 연결. 컴퓨터 안에 "이건 키보드구나!" 말풍선. 중간에 "HID 표준" 박스 표시
- **그림 4.5** `[촬영]` — "PCB 앞면 부위별 명칭"
  - 보드 앞면을 정수직 앵글로 촬영. 후작업으로 USB-C 포트, 리셋 버튼, 버튼 4개, LED, 칩 위치에 화살표와 라벨 오버레이. 흰 배경.
- **그림 4.6** `[촬영]` — "PCB 뒷면"
  - 보드 뒷면을 정수직 매크로 앵글로 촬영. 표면실장 부품이 뚜렷이 보이도록. 흰 배경.
- **그림 4.7** `[일러스트]` — "Pro Micro 호환 핀맵 다이어그램"
  - 표준 Pro Micro 핀맵. GPIO 번호와 특수기능(SDA, SCL, RX, TX) 표시. 우리 키링의 버튼 4개가 D2~D5에 매핑된다고 가정하고, 그 4핀을 색상 강조
- **그림 4.8** `[촬영]` — "USB-C 케이블 — 데이터 가능 vs 충전 전용"
  - 케이블 두 개를 흰 배경에 나란히. 상단 케이블 위에 "데이터 OK" 라벨, 하단 케이블에 "충전 전용 X" 라벨.
- **그림 4.9** `[촬영]` — "노트북에 처음 꽂아본 순간"
  - 노트북 옆 USB-C 포트에 키링이 연결된 모습. 약간 어두운 환경에서 보드 LED가 보이도록. 실제 연결 직후의 자연스러운 장면.
- **그림 4.13** `[스크린샷]` — "드라이버 설치 성공 — 'COM5' 인식"
  - 장치 관리자 "포트(COM 및 LPT)" 항목에 "Arduino Leonardo (COM5)"가 표시된 모습
- **그림 4.10** `[스크린샷]` — "윈도우 장치 관리자 — '알 수 없는 장치'"
  - 장치 관리자 "기타 장치 > 알 수 없는 장치"에 노란 느낌표가 있는 모습
- **그림 4.12** `[스크린샷]` — "드라이버 수동 설치 — .inf 경로 지정"
  - 장치 관리자 우클릭 → 드라이버 업데이트 → 컴퓨터에서 찾기 → Arduino 드라이버 폴더를 선택한 화면

## 7장 채터링과의 전쟁 — 디바운싱

- **그림 7.1** `[촬영]` — "디바운싱 전: 한 번 눌렀는데 'aaaaa' 입력"
  - 메모장에 'aaaaa' 5~10개 + 손가락이 막 첫 번째 버튼에서 떨어진 자세. 위에서 내려다보는 각도.
- **그림 7.2** `[일러스트]` — "기계식 스위치 채터링 파형 (개념도)"
  - 시간 vs 전압 그래프. X축은 시간(ms), Y축은 전압(V). 버튼을 누르는 순간 1~5ms 구간에서 HIGH/LOW가 불규칙하게 진동하다 안정화. 디지털 임계값 라인(점선)을 파형이 여러 번 교차하는 것을 표시.
- **그림 7.3** `[스크린샷]` — "시리얼 모니터로 본 raw digitalRead 값"
  - 시리얼 모니터에 `1/0/0/1/0/0/0/1/1/0...` 빠른 변화가 흘러내려오는 장면. 0과 1이 뒤섞이는 구간에 빨간 박스 표시.
- **그림 7.4** `[스크린샷]` — "시리얼 플로터 — 채터링 그래프"
  - 시리얼 플로터에 그려진 그래프. 대부분 HIGH(1) 수평선이다가 버튼 누르는 순간 LOW/HIGH가 격렬하게 진동. 해당 구간에 빨간 박스 표시.
- **그림 7.5** `[표]` — "디바운싱 두 접근 비교"
  - 비교 항목 / millis() 직접 구현 / Bounce2 라이브러리 - 구현 코드 줄 수: ~20줄 / ~8줄 - 가독성: 중간 (조건 중첩) / 높음 (메서드 이름 직관적) - 버튼 4개 시: 반복 코드 4세트 / 객체 4개 - 학습 가치: 원리 이해에 좋음 / 실무 패턴 습득 - 본 책 채택: — / ✓

- **그림 7.8** `[일러스트]` — "fell() / rose() 동작 시점"
  - 시간 축 위에 버튼 신호 파형(디바운싱 처리 후 깔끔한 구형파). 신호가 HIGH→LOW로 떨어지는 에지에 아래 방향 화살표 + "fell() → true (1 tick)" 레이블. 신호가 LOW→HIGH로 올라오는 에지에 위 방향 화살표 + "rose() → true (1 tick)" 레이블. 그 외 구간에서는 false 표시.
- **그림 7.9** `[표]` — "디바운스 시간 5ms / 10ms / 50ms 비교"
  - 디바운스 시간 / 반응 속도 / 채터링 억제 / 게임 적합성 - 5ms: 매우 빠름 / 약함 (일부 채터링 통과 가능) / 주의 필요 - 10ms: 빠름 / 강함 / 권장 (게임용) - 50ms: 느림 / 매우 강함 / 연타 인식 어려움
