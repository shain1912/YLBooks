# 1부 + 2부 작은 계획표 (1~8장)

## 1장 왜 바이브코딩인가

**쪽수**: 18쪽 / **이미지**: 14장 / **선행**: 없음

### 1장 절 구조
1. **1.1 코딩이 무서웠던 너에게** — 비전공자 대학생이 "코딩은 천재만 한다"는 편견을 깨는 도입부. 저자 본인의 실패담 한 토막과 함께, 2024~2025년 사이 AI 페어 코딩이 어떻게 판을 바꿨는지 짧게 정리.
2. **1.2 바이브코딩이 뭔데?** — "분위기 코딩"이라는 단어의 유래(Karpathy 트윗 등)와 정의. AI에게 의도를 말하면 코드를 받아 다듬는 작업 흐름. 전통적 학습(문법 → 알고리즘 → 프로젝트) vs 바이브코딩(프로젝트 → 막히면 AI에게 묻기) 비교.
3. **1.3 우리가 만들 것 — 키링 게임기 데모** — 책의 최종 결과물(4버튼 리듬게임 키링)을 완성품 사진과 함께 미리 보여줌. 핵심 기능 5가지(USB 꽂으면 인식 / 4버튼 입력 / 채터링 없음 / 리듬게임 OSU에서 작동 / 키캡 커스터마이징).
4. **1.4 학습 로드맵 8주** — 1~3장(도구 설치) → 4~8장(키링 펌웨어) → 9~14장(게임 제작) → 15장(완성)의 8주 코스 시각화. 주당 3~5시간 가정.
5. **1.5 준비물 체크리스트와 예산** — 하드웨어(키링 보드 1.5~2만원, 택타일 스위치 4개, USB-C 케이블, 점퍼선)와 소프트웨어(Claude Code Pro $17~20/월, Antigravity 무료, Arduino IDE 무료)를 표로 정리. 총 예산 5~7만원 선.
6. **1.6 이 책을 읽는 법** — 절 단위 진행 권장, 막히면 5분 룰(5분 이상 헤매면 AI에게 그대로 복붙해서 물어보기), 책 깃허브 저장소 안내, 오탈자 제보 채널.

### 1장 이미지 설명란 (촬영 지시서)
- **그림 1.1** — 캡션: "완성된 4버튼 키링 게임기 — 이 책의 최종 목적지"
  - 촬영 지시: 완성된 키링(투명 아크릴 케이스 + 4개 키캡)을 책상 위 흰 A4 용지 배경에 정중앙 배치. 데스크 램프를 왼쪽 위 45도에서 비춰 키캡의 RGB가 살짝 반사되도록. 카메라는 약 30도 위에서 내려다보는 각도. USB 케이블이 화면 오른쪽으로 살짝 빠져나가게.
- **그림 1.2** — 캡션: "키링이 노트북에 USB로 인식된 모습"
  - 촬영 지시: 노트북 옆모습을 찍되, USB 포트에 키링이 꽂혀 있고 키링이 노트북 옆에 매달린 상태. 노트북 화면에는 장치 관리자 창이 켜져 있어야 함. 자연광 또는 데스크 램프, 그림자 최소화.
- **그림 1.3** — 캡션: "전통적 코딩 학습 vs 바이브코딩 (개념도)"
  - [일러스트: 좌측은 두꺼운 책 더미 위에 사람이 깔린 그림 / 우측은 사람이 노트북 앞에서 AI 말풍선과 대화하는 그림. 화살표는 좌→우 진화 방향]
- **그림 1.4** — 캡션: "Karpathy의 'vibe coding' 원조 트윗 (2024)"
  - [스크린샷: Karpathy의 vibe coding 트윗 원문을 X(트위터) 화면 그대로 캡처. 한국어 번역은 캡션 아래에 별도 박스로]
- **그림 1.5** — 캡션: "키링 게임기로 OSU!를 플레이하는 장면"
  - 촬영 지시: 모니터 화면에 OSU! 게임 플레이 중인 모습이 보이고, 책상 앞쪽에 키링 게임기를 두 손으로 누르는 모습. 손가락 4개가 키캡 위에 자연스럽게 올라간 자세. 모니터 광원만 사용해 게이머스러운 분위기. 셔터 스피드 조금 길게(0.5초 이내)해서 키캡 LED 빛이 살짝 번지게.
- **그림 1.6** — 캡션: "8주 학습 로드맵"
  - [일러스트: 가로축에 1~8주, 세로에 각 주차의 핵심 결과물(도구 설치 / Blink / 1버튼 / 디바운싱 / 4버튼 / 게임 / 케이스 / 완성)을 작은 아이콘과 함께 배치한 타임라인]
- **그림 1.7** — 캡션: "준비물 한눈에 보기"
  - 촬영 지시: 흰 A4 용지 위에 책상 평면샷. 위에서 정수직으로 내려다본 앵글(top-down). 좌측부터 ATmega32U4 키링 보드 1개 / 택타일 스위치 4개 / USB-C 케이블 1개 / 점퍼선 한 묶음 / 가위 / 양면테이프 순서로 줄세워 배치. 각 물품 사이 1cm 간격. 자연광 권장.
- **그림 1.8** — 캡션: "ATmega32U4 키링 보드 클로즈업"
  - 촬영 지시: 키링 보드 단독 클로즈업. 보드의 칩(ATmega32U4 글자가 보이도록)이 화면 중앙에 오게 매크로 모드로. 보드 하단에 자 또는 동전을 같이 배치해 크기 비교 가능하게. 흰 배경.
- **그림 1.9** — 캡션: "도구별 비용 정리표"
  - [일러스트/표: Claude Code Pro $17~20/월, Antigravity 베타 무료, Arduino IDE 무료, 하드웨어 약 3~5만원을 표 형식으로]
- **그림 1.10** — 캡션: "이 책의 깃허브 저장소 첫 화면"
  - [스크린샷: 책 부속 GitHub 저장소 메인 페이지 캡처. README의 챕터별 코드 폴더 구조가 보이도록]
- **그림 1.11** — 캡션: "5분 룰 — 막히면 AI에게 그대로 복붙"
  - [일러스트: 시계가 5분을 가리키는 그림 + 에러 메시지가 AI 채팅창으로 드래그되는 시각화]
- **그림 1.12** — 캡션: "저자가 처음 만든 키링(좌)과 5번째 시도(우)"
  - 촬영 지시: 두 개의 키링을 흰 종이 위에 좌우로 나란히 배치. 좌측은 일부러 어설프게 만든 프로토타입(브레드보드에 점퍼선이 엉킨 상태), 우측은 깔끔하게 마감된 완성품. 같은 조명, 같은 거리에서 촬영해 비교 가능하게.
- **그림 1.13** — 캡션: "필수 소프트웨어 3종 로고"
  - [일러스트: Claude Code, Antigravity, Arduino IDE의 공식 로고 3개를 가로로 나열]
- **그림 1.14** — 캡션: "이 책의 챕터 의존 관계도"
  - [일러스트: 1→2→3→4→5→6→7→8 그리고 8→9 분기되는 노드 그래프. 1부, 2부, 3부, 4부 색상 구분]
